### ⚠️ P0-7:`/news/[slug]` 404 — 文章列表存在但详情页全部 404

**直接证据:** 在公开站测试中,所有 `/news/{slug}` 详情页返回 404(包括截图示例 `brand-website-prep` / `shunde-store-upgrade` / `service-matrix`)。

**截图发现的真因:** 详情页调用 `item.content` 但 `NewsItem` 类型无此字段(commit 0b8f38c)。详见已有 pre-existing bug。

**修复路径:**
1. 在 `src/types/news.ts` 加 `content: string` 字段
2. 在 `prisma/schema.prisma` 加 `content` 列(若用 DB) / 在静态数据中补 `content`
3. `src/app/news/[slug]/page.tsx:94` 引用即可正常

**优先级:** P0(详情页全 404,SEO + 用户体验双输)

### P3-4:所有文章作者 = "系统管理员"

**现象:** 8 篇文章全部 author 显示"系统管理员"。

**问题:**
1. 不能区分是 admin 还是 editor 写的
2. 多 admin 协作时无法归属

**根因推测:** NewsItem schema 没 `authorId` 字段,只存 `authorName`(硬编码为种子用户名)。

**修复:** NewsItem 加 `authorId: string`,从 session.user 取真实 ID。
