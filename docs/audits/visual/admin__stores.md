## 数据发现

### 22 条门店 — 全是草稿

**截图所见:** 所有 22 条记录的状态列均显示"草稿"(灰色徽章)。

**期望:** 至少 1 条应标记为"已发布"或"营业中",因为这是真实在用品牌(顺德大良店)。

**影响:**
1. /api/stores 仍返回所有 22 条(草稿不过滤) → 公开 /agent 列表显示一堆测试数据
2. 截图看到的店名:`深圳前海汽车服务有限...` / `厦门市软件园...` / `Main-st.` / `No 8 main 12...` / `金华汽车 2座` / `TC A1-phonetic 1` / `ES Test 7` / `SS Test 7` × 4 / `ssdd` / `TC ul 1p2 7` / `Plszhonggu 智能 57 6` / `星海四电马蹄医院之家` / `草莓数据 7` / `7试w 7层` / `7电试7 8...` 等 — **明显是开发测试数据**,不是真实业务门店
3. 只有一条疑似真实:`深圳前海汽车服务有限...`(可能是真实,但状态仍草稿)

### ⚠️ P0-6:测试门店数据污染生产 DB

**风险:** 公开 `/agent` 页面显示这些测试门店,SEO 风险 + 用户混淆。

**优先级:** P0(已影响公开站数据)

**修复路径:**
1. **临时:** 后台 /admin/stores 列表 → 批量下架 / 删除所有非真实门店
2. **根治:** Seed 脚本不应包含测试门店 → `prisma/seed.ts` 中删除 store 创建,或加 `status: 'published'` 默认值
3. **未来:** /api/stores 公共查询加 `WHERE status = 'published'` 过滤(避免草稿数据泄露到前台)

### ⚠️ P1-9:店名 ASCII 噪声污染

**现象:** 店名含 `TC A1-phonetic` / `ssdd` / `ES Test 7` / `Plszhonggu` 等中英混杂 / 拼音测试字符串。

**修复:**
1. 删除所有 ASCII 噪声测试数据
2. 新建门店表单加**实时字段校验**(正则:`^[\u4e00-\u9fa5]+$` 或允许数字/字母但禁止纯拼音)

## 与 `/admin/stores/new`(新建)联动

新建表单截图(`admin__stores__new.png`)验证:
- 表单字段:门店名称 / URL 标识(slug) / 省份·城市 / 区域 / 详细地址 / 联系电话 / 营业时间 / 门店描述 / 门店图片 / 营业状态
- 字段标记 `*` 红色 = 必填(5 个必填:门店名称/slug/省·市/详细地址/联系电话)
- 省份·城市是 **级联下拉**(选省后才激活市)
- 营业状态默认 `营业中`
- CTA:取消 / **创建门店**(orange-500)
- 底部有 friendly notice:"选择「下架」后,前台 /api/stores 将不再返回该门店,但后台列表仍可见。"

**表单 UX:** 优秀,信息架构清晰,字段示例充足。
