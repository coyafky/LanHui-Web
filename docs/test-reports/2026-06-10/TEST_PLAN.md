## 1. 测试方法

| 方法 | 工具 | 适用范围 |
|------|------|----------|
| **浏览器真实访问** | Chrome DevTools MCP / Playwright | 所有前端页面的渲染、交互 |
| **API 直接调用** | curl + JSON | 验证后端逻辑 |

| **数据库直查** | psql / Prisma Studio | 数据一致性验证 |

## 5. 验收清单

### 5.1 必做测试（每个页面）

- [ ] 页面能正常访问（无 500 错误）
- [ ] 关键内容正确渲染
- [ ] 主要交互（链接/按钮/表单）可正常工作
- [ ] 响应式布局正常（mobile/tablet/desktop）
- [ ] 无明显的视觉错位

### 5.2 必做测试（关键交互）

- [ ] 登录流程
- [ ] 文章 CRUD
- [ ] 状态机切换（草稿 ↔ 已发布 ↔ 已归档）
- [ ] 批量操作
- [ ] 分页
- [ ] 筛选 + 搜索
- [ ] 弹窗确认

### 5.3 必做测试（API）

- [ ] 未认证访问受保护端点 → 401
- [ ] 输入验证（Zod）正确
- [ ] 数据库读写一致

## 6. 缺陷严重度定义

| 级别 | 含义 | 处置 |
|------|------|------|
| **P0** | 阻断：核心功能不可用 | 立即修复 |
| **P1** | 严重：主要功能异常 | 24h 内修复 |
| **P2** | 一般：次要功能问题 | 排期修复 |
| **P3** | 轻微：体验/文案/样式 | 有空再修 |
