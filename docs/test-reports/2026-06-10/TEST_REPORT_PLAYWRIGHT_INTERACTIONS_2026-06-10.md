- ❌ **【P1-Bug-004】操作菜单点击不打开**：点击每行末尾的「三点」按钮，下拉菜单**未显示**

**Bug-004 详情**:
- **现象**: 点击文章行的操作菜单按钮（MoreHorizontal 图标），下拉菜单不出现
- **重现步骤**:
  1. 访问 `/admin/articles`
  2. 点击任意文章行末尾的「三点」按钮
  3. 期望：弹出「编辑/回退到草稿/删除」菜单
  4. 实际：菜单立即关闭
- **根因**（推测）: `articles/page.tsx:113-119` 的 `document.addEventListener("click", ...)` 全局关闭菜单监听器与按钮的 `e.stopPropagation()` 冲突。React 17+ 事件委托机制下，native document 监听器在 React 合成事件之后触发，导致菜单打开后立即被关闭。
- **影响**: 用户无法通过 UI 编辑/回退/删除单篇文章

**结论**: ⚠️ 列表渲染/筛选/批量选择正常，但**单篇操作菜单不可用**

### 3.4 编辑文章 `/admin/articles/[id]`

**步骤**:
1. `playwright-cli goto http://localhost:3000/admin/articles/cmq7ncbz2001p4vg6zwtow1mh`
2. 验证表单预填

**实际结果**:
- ❌ **【P1-Bug-008】表单未预填**：所有字段（标题/slug/描述/内容/图片/分类/标签）全部空白
- ⚠️ 状态字段显示「draft」是默认值（默认状态），非加载数据
- ❌ **【P0-Bug-009】API 返回 404**：通过 auth fetch 验证：`/api/articles/cmq7ncbz2001p4vg6zwtow1mh?preview=true` 返回 `{success: false, error: "文章不存在"}`

**Bug-009 详情（根因）**:
- **文件**: `src/app/api/articles/[id]/route.ts:15`
- **代码**: `const isCuid = id.startsWith("cl") && id.length > 20;`
- **问题**: 硬编码 CUID 前缀为 "cl"，但 Prisma 生成的 cuid 前缀是 "cm"（"c" + "m"）等
- **影响**: 所有现有文章（id 均以 "cm" 开头）都无法被该 API 识别为 cuid，被错误地按 slug 查询，导致全部返回 404
- **修复建议**: 改为 `id.length === 25` 或正则匹配 cuid 格式

**Bug-008 详情**:
- **现象**: 编辑页表单未预填文章数据
- **根因**: API Bug-009 导致 `loadArticle()` 获取数据失败，状态未被设置
- **影响**: 编辑功能完全不可用（连加载都失败）

**结论**: ❌ 编辑功能**完全不可用**（2 个 P0/P1 Bug 阻断）

## 4. 发现的问题汇总

### 4.1 P0 阻断级

#### [P0] Bug-009: CUID 格式检测错误导致所有文章编辑/预览失败
- **文件**: `src/app/api/articles/[id]/route.ts:15`
- **代码**: `const isCuid = id.startsWith("cl") && id.length > 20;`
- **现象**: 编辑文章/预览草稿时 API 返回 404 "文章不存在"
- **根因**: cuid 前缀硬编码为 "cl"，但实际数据为 "cmq7..."
- **影响范围**: 所有文章编辑、预览功能
- **修复建议**: 改为 `id.length === 25` 或正则 `/^c[a-z0-9]{24}$/`

### 4.2 P1 严重级

#### [P1] Bug-004: 文章操作菜单点击不打开
- **文件**: `src/app/admin/(dashboard)/articles/page.tsx:113-119`
- **现象**: 点击 MoreHorizontal 按钮，下拉菜单不显示
- **根因**: document 级别 click 监听器与 React stopPropagation 冲突
- **影响**: 用户无法编辑/回退/删除单篇文章

#### [P1] Bug-008: 编辑页表单未预填（Bug-009 的连锁反应）
- **文件**: `src/app/admin/(dashboard)/articles/[id]/page.tsx:48-76`
- **现象**: 访问编辑页，所有字段空白
- **根因**: API Bug-009 阻断数据加载
- **影响**: 编辑功能完全不可用

#### [P1] Bug-007: 文章列表无可见的编辑入口
- **文件**: `src/app/admin/(dashboard)/articles/page.tsx`
- **现象**: 列表中文章标题不是链接，操作菜单（Bug-004）也无法打开
- **影响**: 普通用户**无法编辑任何文章**

### 4.3 P2 一般级

#### [P2] Bug-001 (历史): ISR 缓存导致新文章不会立即在 /news 第 1 页显示
- **来源**: 上一轮测试报告
- **建议**: 后台发布/编辑文章时调用 `revalidatePath('/news')`

### 4.4 验证结果

#### ✅ 非 Bug：API 状态过滤「正常」工作
- curl 终端调用 `/api/articles?status=draft` 返回 8 篇 published 是**预期行为**
- 公共 API 只返回 published；admin 端通过 session 判断
- 详情见 `src/app/api/articles/route.ts:36-40`
