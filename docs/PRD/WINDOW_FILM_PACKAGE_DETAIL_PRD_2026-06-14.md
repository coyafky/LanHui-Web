## 1. 背景

当前 `/product/window-film` 已经展示了“太阳膜套餐推荐”和“单品参数一览”，但内容偏参数化：

- 用户能看到套餐名、前挡/侧后挡搭配、质保和参数。
- 用户还不知道每个套餐为什么适合自己。
- 用户难以把参数转换成真实用车体验。
- 每个套餐没有独立页面，不利于后续 SEO、销售讲解、企业微信客服引用。

因此需要补齐两层内容：

1. 在汽车窗膜总页增加“用户看得懂的产品话术模块”。
2. 为每个太阳膜套餐创建详情页，用于解释套餐作用、适合人群、使用场景、案例效果、参数解读和施工交付。

## 2. 用户目标

用户进入汽车窗膜页时，真实问题通常不是“哪个参数最高”，而是：

- 夏天车里太晒，哪套更凉快？
- 前挡贴膜会不会影响视线？
- 后排想要隐私，该选哪套？
- 新能源车玻璃面积大，哪套更适合？
- 家里有小孩，应该优先看防晒还是隐私？
- 预算有限，哪套够用？
- 高端车或新车，哪套更稳妥？
- 这些参数到底和真实体验有什么关系？

页面需要把“膜参数”翻译成“用车收益”。

## 3. 产品目标

### 3.1 总页目标

`/product/window-film` 继续作为汽车窗膜总览页，新增内容模块：

- 用户痛点模块：晒、热、眩光、隐私、紫外线、玻璃安全。
- 参数解释模块：VLT、UVR、IRR、TSER、厚度分别意味着什么。
- 套餐选择导购模块：按用户场景推荐套餐。
- 套餐卡片增加“查看详情”入口。
- 保留现有参数表，但不要让参数表成为唯一说服内容。

### 3.2 套餐详情页目标

为每个套餐生成独立详情页：

```text
/product/window-film/chunfen
/product/window-film/guyu
/product/window-film/xiaoman
/product/window-film/mangzhong
/product/window-film/bailu
/product/window-film/wanghong
/product/window-film/yangsheng
```

每个详情页需要回答：

- 这个套餐适合什么样的车主？
- 解决哪些实际问题？
- 前挡和侧后挡为什么这样搭配？
- 施工后用户能感受到什么变化？
- 有哪些使用场景案例？
- 参数如何理解？
- 施工和验收需要注意什么？

## 6. 总页新增模块

### 6.1 用户痛点模块

标题建议：

```text
贴太阳膜，不只是为了隔热
```

模块内容：

| 痛点 | 用户语言 | 页面话术 |
| --- | --- | --- |
| 热 | 夏天上车像进烤箱 | 通过前挡和侧后挡组合降低阳光热量进入车内，减轻空调负担。 |
| 晒 | 皮肤和内饰长期暴晒 | 高紫外线阻隔有助于减少皮肤和内饰被阳光长期伤害。 |
| 眩光 | 逆光、午后开车刺眼 | 合理前挡透光率可以兼顾清晰视野和强光舒适度。 |
| 隐私 | 后排和车内物品容易被看到 | 侧后挡搭配隐私膜，提升车内私密性。 |
| 安全 | 玻璃破裂飞溅 | 膜层可提升玻璃破裂时的附着能力。 |
| 新能源 | 玻璃面积大，热量更明显 | 新能源车型前挡和天幕面积大，更需要系统化隔热方案。 |

### 6.2 参数翻译模块

标题建议：

```text
参数怎么选，先看这几个指标
```

内容：

| 参数 | 全称/含义 | 用户理解 |
| --- | --- | --- |
| VLT | 可见光透过率/阻隔率，项目内当前字段叫“可见光阻隔率” | 影响视觉明暗和隐私感，前挡应优先保证清晰视线。 |
| UVR | 紫外线阻隔率 | 关系到防晒、内饰老化、皮肤舒适度。 |
| IRR | 红外线阻隔率 | 关系到体感热量，红外线越高通常越抗晒热。 |
| TSER | 总太阳能阻隔率 | 综合隔热指标，更接近整体隔热表现。 |
| 厚度 | 膜层厚度 | 影响施工手感、安全附着和产品定位，不应单独作为唯一判断标准。 |

注意：

- 当前数据字段写的是“可见光阻隔率”，但行业常见表达是可见光透过率 VLT。架构师需确认品牌资料真实口径，避免字段含义写错。
- 如果无法确认，页面上不要扩写为“透过率”，只解释为“当前资料口径下的可见光指标”。

### 6.3 选择导购模块

标题建议：

```text
按你的用车需求选套餐
```

推荐逻辑：

| 用户需求 | 推荐套餐 | 推荐话术 |
| --- | --- | --- |
| 健康环保、防紫外线、隐私入门 | 春分套餐 | 入门友好，前挡清晰，侧后挡兼顾隐私。 |
| 想要比入门更强的综合隔热 | 谷雨套餐 | 兼顾紫外线、普通隔热和隐私，适合日常通勤。 |
| 想要全车隔热和安全防护 | 小满套餐 | 前挡 12 层金属膜搭配侧后挡，适合对隔热和安全更重视的车主。 |
| 更强调侧后挡隔热和隐私 | 芒种套餐 | Z70 + Z20 组合，适合家庭车和新能源车。 |
| 想要前挡变色效果 | 白露套餐 | 前挡 Z80 有视觉变化，适合追求科技感和高级感的车主。 |
| 喜欢高辨识度颜色 | 网红套餐 | 帝王紫/凤凰红风格明显，适合个性表达。 |
| 强调健康、环保、舒适 | 养生套餐 | 负离子膜组合，适合家庭用户和重视车内体验的车主。 |

## 7. 套餐详情页结构

每个套餐详情页统一使用以下结构。

### 7.1 Hero

内容：

- 套餐名
- 一句话定位
- 适合人群
- 前挡 + 侧后挡组合
- 质保信息

示例：

```text
春分套餐
健康环保入门选择，适合重视防晒、隐私和清晰视线的日常车主。
前挡：环保陶瓷膜 K7
侧后挡：环保陶瓷膜 C15
质保：5年
```

### 7.2 场景痛点

每个套餐需要 3-4 个场景痛点。

示例：

- 午后暴晒后车内升温明显。
- 后排儿童或家人需要更舒适的防晒。
- 日常通勤想减少强光干扰。
- 停车时希望车内物品不那么容易被看到。

### 7.3 套餐作用

用用户语言解释套餐作用：

- 前挡负责视线清晰与正面隔热。
- 侧后挡负责隐私、防晒和后排舒适。
- 前后组合避免“前挡清楚但侧后太晒”或“侧后够暗但前挡不舒服”的割裂体验。

### 7.4 参数解读

每个详情页都需要一张参数解读表：

| 位置 | 产品 | 参数 | 用户理解 |
| --- | --- | --- | --- |
| 前挡 | K7 | 原始参数 | 前挡要保持清晰视野，同时阻隔热量和紫外线。 |
| 侧后挡 | C15 | 原始参数 | 侧后挡更偏隐私和后排舒适。 |

### 7.5 场景案例

注意：如果没有真实客户授权图片或真实案例，不得写成“真实客户案例”。

建议统一命名为：

```text
典型用车场景
```

每个套餐 2-3 个场景：

- 家庭通勤
- 新能源车玻璃面积大
- 高速长途
- 后排儿童
- 商务接待
- 年轻个性外观

### 7.6 效果展示

首版可以使用占位图片或生成图片，但要明确规格。

建议展示：

- 车外侧后挡隐私效果
- 车内前挡清晰视野
- 阳光照射下的隔热/防晒场景
- 施工完成后的整体质感

### 7.7 施工与验收

复用现有膜系服务保障内容：

- 车型确认
- 玻璃清洁
- 裁膜/烤膜
- 装贴排水
- 边缘检查
- 交付提醒

验收重点：

- 无明显折痕
- 无明显气泡
- 无影响视线的瑕疵
- 边缘贴合
- 升降窗提醒

谷雨套餐把前挡金属膜和侧后陶瓷护肤膜组合起来，适合关注防晒、普通隔热和隐私的车主。它不是最激进的高隔热路线，而是偏向日常舒适和稳定体验。

适合：

export type ProductPackage = {

## 10. 路由与组件建议

### 10.1 新增路由

```text
src/app/product/window-film/[packageSlug]/page.tsx
```

页面能力：

- `generateStaticParams()` 生成 7 个套餐详情页。
- `generateMetadata()` 根据套餐名生成 title/description。
- 找不到套餐时 `notFound()`。

### 10.2 新增组件

建议组件：

```text
src/components/window-film/WindowFilmGuide.tsx
src/components/window-film/WindowFilmPackageCard.tsx
src/components/window-film/WindowFilmPackageDetail.tsx
src/components/window-film/WindowFilmParameterExplainer.tsx
src/components/window-film/WindowFilmScenarioGrid.tsx
```

组件边界：

- `WindowFilmGuide`：总页新增导购模块。
- `WindowFilmPackageCard`：替换当前 `ProductDetail` 内联套餐卡片，增加详情链接。
- `WindowFilmPackageDetail`：套餐详情页主体。
- `WindowFilmParameterExplainer`：参数解释，可在总页和详情页复用。
- `WindowFilmScenarioGrid`：典型用车场景。

### 10.3 是否继续放在 ProductDetail

建议：

- 不要继续把所有窗膜详情页逻辑塞进 `ProductDetail.tsx`。
- `ProductDetail.tsx` 已经承担 PPF、窗膜、改色膜多个条件分支，继续膨胀会降低可维护性。
- 窗膜套餐详情页应使用独立组件目录。

## 11. 视觉与交互要求

设计方向：

- 保持当前深色品牌风格。
- 比现有卡片更有层次，但不要做成营销落地页。
- 卡片圆角不超过现有系统风格。
- 表格在移动端必须横向可滚动或转为信息块。
- 套餐详情页首屏需要比总页更有“产品专题感”。
