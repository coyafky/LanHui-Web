# 蓝辉咨询渠道后台管理 PRD

> **系统范围**：`/admin/consultation-channels` 及相关 API、二维码上传和操作日志  
> **公开站契约**：[CONSULTATION_CHANNEL_SYSTEM_PRD_2026-06-22.md](../public-site/CONSULTATION_CHANNEL_SYSTEM_PRD_2026-06-22.md)  
> **用户**：总部 admin  
> **版本**：v0.1  
> **状态**：规划中，未授权编码

## 1. 目标

让总部员工无需修改代码即可管理官网使用的微信、企业微信和电话咨询渠道，并安全控制二维码从准备、启用到停用的生命周期。

## 2. 权限

| 能力 | admin | editor |
|---|---:|---:|
| 查看渠道 | 是 | 可选只读 |
| 新建和编辑 | 是 | 否 |
| 上传/替换二维码 | 是 | 否 |
| 启用和停用 | 是 | 否 |
| 设为总部默认 | 是 | 否 |
| 查看操作日志 | 是 | 否 |

## 3. 状态

| 状态 | 含义 | 官网可用 |
|---|---|---:|
| `draft` | 正在准备或未完成校验 | 否 |
| `active` | 当前可用于咨询承接 | 是 |
| `disabled` | 已停用但保留历史 | 否 |

状态流：

```text
draft → active → disabled
  ▲         │
  └─────────┘
```

重新启用 disabled 渠道前必须重新检查二维码、承接主体和生效时间。

## 4. 页面

### `/admin/consultation-channels`

- 搜索渠道名称。
- 按渠道类型、范围、门店和状态筛选。
- 显示二维码缩略图、承接主体、状态、优先级和更新时间。
- 提供新建、编辑、预览、启用和停用动作。

### `/admin/consultation-channels/new`

- 新建 draft。
- 选择渠道类型和适用范围。
- 上传二维码。
- 保存后预览公开弹窗。

### `/admin/consultation-channels/[id]`

- 编辑资料。
- 替换二维码。
- 设置总部默认或门店归属。
- 设置生效时间。
- 启用、停用和重新启用。
- 查看操作记录。

## 5. 字段

| 字段 | draft 必填 | active 必填 |
|---|---:|---:|
| 内部名称 | 是 | 是 |
| 渠道类型 | 是 | 是 |
| 承接主体展示名 | 否 | 是 |
| 范围 `headquarters/store` | 是 | 是 |
| 门店 | 范围为 store 时 | 范围为 store 时 |
| 二维码图片 | 否 | 微信类必填 |
| 账号提示 | 否 | 否 |
| 联系电话 | 否 | phone 类型必填 |
| 优先级 | 否 | 是，默认 100 |
| 生效时间 | 否 | 否 |
| 失效时间 | 否 | 否 |
| 内部备注 | 否 | 否 |

“账号提示”只用于公开展示经过确认的账号名称，不强制显示个人微信号。

## 6. 业务约束

- 一期同一时间最多一个总部默认企业微信。
- 门店专属渠道必须关联 active 或可恢复的真实 Store。
- 门店被 suspended/terminated 后，其专属渠道不能继续公开解析。
- 微信类渠道启用前必须有二维码图片。
- 结束时间早于开始时间时禁止保存。
- 停用渠道需要填写原因并记录操作日志。
- 替换二维码成功后再删除旧文件，失败时保留旧二维码。

## 7. 二维码管理

- JPG、PNG、WebP，最大 5MB。
- 服务端验证真实图片格式。
- 建议保持正方形，转为 WebP。
- 后台显示大图预览和扫描测试提示。
- 支持替换，不建设通用媒体库。
- 文件路径由服务端生成，不接受用户控制的文件名。

## 8. 预览

后台预览应复用公开站统一咨询弹窗，显示：

- 桌面和移动宽度。
- 承接主体。
- 渠道类型。
- 二维码。
- 降级电话。
- 一个模拟来源场景。

预览不产生公开站 `contact_click`。

## 9. 异常

| 场景 | 处理 |
|---|---|
| 二维码上传失败 | 保留旧图和表单内容 |
| 启用校验失败 | 保持原状态并列出缺失字段 |
| 默认渠道冲突 | 事务内切换或阻止重复默认 |
| 门店不存在 | 阻止保存 |
| 渠道正在被公开使用 | 允许停用，但明确提示将触发回退 |
| 数据库失败 | 不显示虚假成功 |

## 10. 验收

- [ ] admin 能完成 draft、active、disabled 生命周期。
- [ ] 启用前完成二维码和归属校验。
- [ ] 同一时间只有一个总部默认企业微信。
- [ ] 替换失败不破坏当前有效二维码。
- [ ] 停用后公开站按规则回退。
- [ ] 所有写操作记录 ActivityLog。
