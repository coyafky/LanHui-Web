# BRAND_PRD_2026-06-20.md — 品牌子站 `/brand` 系列完整规格

> 蓝辉轻改 LANHUI 品牌子站 v1 规格文档。覆盖 `/brand`(品牌故事)、`/brand/certifications`(资质证书)、`/brand/history`(发展历程)3 个公开路由,统一吸收 v0 `LOGO_BRAND_VISUAL_ALIGNMENT_PRD_2026-06-14.md` 的品牌视觉规范(Logo 资产、Header/Footer 适配、Schema.org logo 路径)。

---

## 1. 概述

**页面**: `/brand`、`/brand/certifications`、`/brand/history`
**类型**: 公开站(SSG, RSC)
**优先级**: P0
**Owner**: 冯科雅(Coya)
**版本**: v1
**最后更新**: 2026-06-20

### 1.1 目标

1. 让潜客在 3 分钟内建立"蓝辉轻改是正规、源头、专业、有沉淀"的品牌信任。
2. 用资质证书和发展历程两个子页分别承载"合规凭证"和"时间轴叙事"两个信任维度。
3. 统一品牌子站视觉语言(深色 + 蓝橙 + rounded-2xl 卡片),与公开站其他页面保持一致。
4. 沉淀 Logo 资产规范,作为后续物料 / 后台 / 多平台的源头。

### 1.2 范围

- ✅ 包含: 3 个公开路由 + 共享 Header / Footer + Logo 组件视觉规范
- ✅ 包含: 资质证书数据结构(基于 `src/lib/certifications.ts`)、发展历程时间轴(`src/lib/history.ts`)
- ✅ 包含: v0 Logo 视觉规范的吸收(资产目录、Header/Footer 适配、Schema.org 路径)
- ❌ 不包含: 完整 VI 手册 / 招商物料 / 品牌动态视频
- ❌ 不包含: 营业执照原件 PDF 在线展示(法律风险)

## 4. UI / 交互

### 4.1 视觉规范(沿用公开站)

- **背景**: `bg-zinc-950` / `bg-black` / `bg-zinc-900`(卡片)
- **文字**: `text-white` / `text-zinc-300` / `text-zinc-400` / `text-zinc-500`
- **品牌色**: `orange-400/500` 强调 + `blue-400/500/700` 科技
- **里程碑徽标**: `bg-orange-500/10 border border-orange-700/50 text-orange-300`
- **资质分类图标**: `Award` icon `text-orange-400`
- **字体**: Geist Sans + 系统中文
- **圆角**: 卡片 `rounded-2xl`,徽标 `rounded-md / rounded-xl`

### 4.2 组件清单

| 组件 | 路径 | 类型 | 说明 |
|---|---|---|---|
| `Header` | `src/components/Header.tsx` | CC | 共享 |
| `Footer` | `src/components/Footer.tsx` | RSC | 共享 |
| `Logo` | `src/components/Logo.tsx` | RSC | 透明底 / 滚动缩放(沿用 v0 §9.1) |
| `CertCard` | `src/components/CertCard.tsx` | RSC | 单个证书卡片(占位图 + 名称) |
| `TimelineDot` | (内联于 `history/page.tsx`) | RSC | 时间轴圆点 |
| `MilestoneCard` | (内联于 `history/page.tsx`) | RSC | 单个时间节点卡片 |

### 4.3 路由结构

| 路由 | 文件 | metadata |
|---|---|---|
| `/brand` | `src/app/brand/page.tsx` | `title: "品牌介绍 \| 蓝辉轻改 LANHUI"` |
| `/brand/certifications` | `src/app/brand/certifications/page.tsx` | `title: "资质证书 \| ..."` |
| `/brand/history` | `src/app/brand/history/page.tsx` | `title: "品牌历程 \| ..."` |

### 4.4 区块级结构

**`/brand` 区块**:
1. Hero: 渐变背景 + ABOUT 标签 + H1 + 副标
2. About(关于蓝辉轻改): 3 段正文,关键名词橙色/蓝色高亮
3. Values(品牌理念): 3 列卡片(Sparkles / Wrench / Car icons)
4. 起点: 2 列卡片(Calendar + MapPin)

**`/brand/certifications` 区块**:
1. Hero: 渐变背景 + 面包屑(品牌介绍 › 资质证书) + CERTIFICATIONS 标签 + H1
2. 4 大资质分类入口(2/4 列响应式)
3. 证书网格(`CertCard` × N,1/2/3 列响应式)
4. "关于证书核验" 提示段(强调"以顺德大良店现场出示为准")

**`/brand/history` 区块**:
1. Hero: 渐变背景 + 面包屑(品牌介绍 › 品牌历程) + MILESTONES 标签 + H1
2. 时间轴 `<ol>`: 桌面左右交替 / 移动单列,中心 spine 渐变线
3. "未完待续" 引导区(Sparkles icon + 跳 /news)

### 4.5 三视口响应式

| 视口 | `/brand` | `/brand/certifications` | `/brand/history` |
|---|---|---|---|
| Desktop 1440 | 3 列 Values / 2 列起点 | 4 列分类 / 3 列证书 | 时间轴左右交替 |
| Tablet 768 | 2 列 Values | 2 列分类 / 2 列证书 | 时间轴左右交替(间距收紧) |
| Mobile 390 | 全单列 | 2 列分类 / 1 列证书 | 时间轴左侧单列,dot 居左 |

### 4.6 可访问性

- 时间轴使用 `<ol>` + `<article>`,语义化
- "里程碑" 用 `<span>` 徽标,文字表达高亮
- 面包屑使用 `<nav>` 包裹
- 颜色对比度 ≥ 4.5:1
- 所有图片有 `alt`(`CertCard` 的占位图 alt 必填)

## 5. 数据模型

### 5.1 静态数据

| 文件 | 内容 | 引用方 |
|---|---|---|
| `src/lib/brand.ts` | 品牌名 / slogan / 成立年 / 城市 / 占位 | 3 路由 |
| `src/lib/certifications.ts` | `certifications[]` + `certCategories[]` | `/brand/certifications` |
| `src/lib/history.ts` | `milestones[]` | `/brand/history` |
| `src/lib/schema.ts` | `organization.logo` URL 指向 `lanhui-logo-light.png` | 全站 JSON-LD |

### 5.2 证书数据结构(参考)

```ts
// src/lib/certifications.ts (推断)
type Certification = {
  id: string;
  name: string;
  category: "business" | "industry" | "store" | "partnership";
  description: string;
  imageUrl?: string;       // 占位图(可选,不上传原件)
  issuedBy?: string;
  issuedAt?: string;
};

type CertCategory = {
  id: string;
  label: string;
  description: string;
};
```

### 5.3 历程数据结构(参考)

```ts
// src/lib/history.ts (推断)
type Milestone = {
  year: string;          // "2026" / "2025" / "2024" ...
  month?: string;        // "1月" / "Q2" (可选)
  title: string;
  description: string;
  highlight?: boolean;   // true → 显示"里程碑"徽标
};
```

### 5.4 Logo 资产规范(吸收 v0 §6)

```text
public/images/logo/
  lanhui-logo-light.svg        # 深色背景横版(Header / Footer)
  lanhui-logo-light.png        # 透明底 1600x400 兜底
  lanhui-logo-dark.svg         # 浅色背景横版
  lanhui-logo-dark.png
  lanhui-logo-color.svg        # 彩色标准版(品牌页)
  lanhui-logo-color.png
  lanhui-symbol.svg            # 仅图形 favicon / 头像
  lanhui-symbol.png            # 512x512
  favicon.svg
  apple-touch-icon.png         # 180x180
```

> v1 至少保证 `lanhui-logo-light.{svg,png}` + `lanhui-symbol.{svg,png}` + `favicon.svg` 就位。

### 5.5 Logo 组件 API(沿用 v0 §9.1)

```ts
type LogoVariant = "light" | "dark" | "color" | "symbol";
type LogoLockup = "full" | "zh" | "symbol";

type LogoProps = {
  variant?: LogoVariant;
  lockup?: LogoLockup;
  className?: string;
  priority?: boolean;
};
```

### 5.6 Logo 尺寸规范(吸收 v0 §8.1)

| 状态 | 高度 |
|---|---|
| 未滚动桌面 Header | 32-36px(实际 `h-10` = 40px,待收紧) |
| 滚动后桌面 Header | 28-32px(实际 `h-8` = 32px ✓) |
| 移动端 Header | 28-32px(实际 `h-8` ✓) |
| Footer | 28-36px(实际 `h-8` ✓) |

### 5.7 SSR / ISR 配置

- 3 路由默认 SSG(`force-static`)
- `/brand` 不设 `revalidate`(内容相对稳定)
- `/brand/certifications` 不设 `revalidate`
- `/brand/history` 不设 `revalidate`
- 部署时重建(Build time SSG)
