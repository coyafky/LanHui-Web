## 1. 产品体系重新定义

产品中心不再只是“六张产品卡 + 四张专题 Banner”，而划分为三个用户能理解的系统：

```text
产品中心
├── 按车型品牌找方案
│   ├── 极氪
│   ├── 小米汽车
│   ├── 问界
│   └── 理想汽车
├── 按项目找服务
│   ├── 电动踏板
│   ├── 轮毂
│   ├── 底盘护板
│   ├── 内饰改装
│   └── 洗美项目
└── 按膜系需求找产品
    ├── 汽车窗膜
    ├── 改色膜
    └── 隐形车衣
```

三者不是重复：

- 品牌页回答“我的车能改什么”。
- 轻改项目页回答“这个改装项目怎么选”。
- 膜系页回答“这么多膜产品如何按需求筛选”。
- 洗美页回答“车辆当前状态适合哪种清洁、美容或养护服务”。

## 3. 产品中心具体模块

### A. Hero

- 标题：`按车型，或按升级需求，找到适合你的方案`
- 说明四个新能源品牌、四类轻改项目、洗美服务和三类膜系服务。
- 不强调产品总数量。

### B. 四个品牌车型入口

每个品牌卡展示：

- 品牌名称。
- 当前覆盖车型。
- 本品牌主要改装方向。
- 一张已确认代表图。
- `查看车型方案`。

| 品牌 | 当前页面重点 |
|---|---|
| 极氪 | 座舱、尾箱、底盘防护、内饰便利 |
| 小米 | 外观风格、轮毂、内饰细节、日常保护 |
| 问界 | 电动踏板、底盘护板、地板尾箱、内饰便利 |
| 理想 | 家庭便利、地板总成、内饰空间、电动踏板 |

### C. 五类项目与服务

- 电动踏板。
- 轮毂。
- 底盘护板。
- 内饰改装。
- 洗美项目。

轻改卡展示“解决什么问题、适合什么车型、如何确认适配”；洗美卡展示“适合什么车况、包含哪些服务、如何确认门店”，而不是只展示图标。

### D. 膜系产品选择入口

汽车膜款式多，不在产品中心平铺几百个型号，而先按需求分流：

- 隔热与隐私 → 汽车窗膜。
- 外观颜色表达 → 改色膜。
- 漆面保护 → 隐形车衣。

### E. 方案选择说明

解释车型、年款、配置、预算和使用场景会共同影响选择，最终需门店确认。

### F. 文章与门店

- 最新产品知识。
- 查找真实门店。
- 企业微信咨询。

## 4. 页面间关联

示例：

```text
问界 M9 品牌页
├── 电动踏板 → 电动踏板项目页
├── 底盘护板 → 底盘护板项目页
├── 地板/坐垫 → 内饰改装页
└── 咨询 → 带 wenjie/M9 上下文
```

每个品牌页至少关联四类轻改项目中实际存在的项目；不存在或未确认的项目不为了版面完整强行展示。

- [ ] 品牌页和项目页双向链接。
