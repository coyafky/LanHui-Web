## 1. 概述

**页面**: `/news`、`/news/[slug]`

### 1.1 目标

1. 修复 P0-7:`/news/[slug]` 8 条已发布文章详情全部 404,SEO 与用户分享链接死链。
2. 提供 8 条文章(目前 3 条静态 + 5 条 DB)详情页完整渲染,带 SEO 友好 metadata、Related 区块、面包屑。
3. 列表页分页(每页 5 条)+ 分类过滤(品牌 / 门店 / 产品)扩展点预留。
4. 详情页正文渲染统一走 `ArticleContent` 组件(已存在),支持段落 / 标题 / 列表基础排版。
5. 类型与数据双锁,防止"加了 `item.content` 引用但没补字段"再次发生。

### 1.2 范围

- ✅ 包含: 2 路由 + 完整 SEO / a11y / 性能 / 埋点

## 5. 数据模型

### 5.1 NewsItem 类型(当前 + v1 修复)

**当前**(有 bug,`content` 标记 optional 但 `src/app/news/[slug]/page.tsx:94` 直接读):

```ts
// src/lib/news.ts (commit 0b8f38c 之前)
export type NewsItem = {
  slug: string;
  title: string;
  date: string;
  category: string;
  summary: string;
  content?: string;   // optional → 详情页读不到时为 undefined
};
```

**v1 修复目标**(必填 + 全文非空 + Zod 校验):

```ts
// src/lib/news.ts (v1)
export type NewsItem = {
  slug: string;
  title: string;
  date: string;            // ISO YYYY-MM-DD 或 "2026" 占位
  category: string;        // "品牌动态" | "门店动态" | "产品动态"
  summary: string;         // 1-2 句摘要(列表用)
  content: string;         // 必填,≥ 200 字正文
  authorId?: string;       // 可选,关联 User.id(P2-4b 准备字段)
  coverImageUrl?: string;  // 可选,列表/详情 OG 图
  tags?: string[];         // 可选,SEO 关键词
};
```

### 5.2 静态 + DB 双源

| 来源 | 数量 | 用途 |
|---|---|---|
| `src/lib/news.ts` `newsItems` | 3 条 | 静态兜底(Build 通过不依赖 DB) |
| DB `Article` 表 | 5+ 条 | 真实运营文章(已发布 status='published') |
| `src/lib/data.ts` `getArticles()` | 聚合 | API 优先 + static fallback |

### 5.3 Article DB Schema(参考 [../../database/SCHEMA.md](../../database/SCHEMA.md) §Article)

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `String` (cuid) | 主键 |
| `slug` | `String` (unique) | URL slug |
| `title` | `String` | 标题 |
| `summary` | `String` | 摘要 |
| `content` | `String` | **正文(必填,Markdown/HTML)** |
| `category` | `String` | 分类枚举 |
| `status` | `String` | `"published"` / `"draft"` / `"archived"` |
| `authorId` | `String?` | 外键 → `User.id` |
| `coverImageUrl` | `String?` | 列表/OG 图 |
| `publishedAt` | `DateTime?` | 发布时间(用于排序) |
| `createdAt` / `updatedAt` | `DateTime` | — |

### 5.5 DB 5 条文章 content 补全

- 用 `npx tsx scripts/seed-content.ts` 或 prisma studio 给 5 条已发布文章补 content(每条 ≥ 200 字正文)
- content 字段类型 `Text`(无长度限制)
- seed script 见 v1 实施任务

### 5.6 SSR / ISR 配置

```ts
// src/app/news/page.tsx
export const revalidate = 3600;   // 1h ISR

// src/app/news/[slug]/page.tsx
export const revalidate = 3600;
```

- `generateStaticParams` 调 `getAllArticleSlugs()` 预生成所有详情
- `generateMetadata` 按 slug 生成独立 title / description
- 新文章发布后,最长 1h 后 ISR 重建(也可用 `revalidatePath('/news/[slug]', 'page')` 主动触发)

### 5.7 SSG 流程

```
build time:
  1. 调 getAllArticleSlugs() → 8 个 slug
  2. 对每个 slug,预渲染 /news/<slug>
  3. 每条详情页生成 metadata + JSON-LD

runtime:
  - 命中预渲染 → 直接返回(0 ms)
  - 未命中(新文章)→ 走 ISR(最长 1h)→ 调 getArticleBySlug() → 缓存
```
