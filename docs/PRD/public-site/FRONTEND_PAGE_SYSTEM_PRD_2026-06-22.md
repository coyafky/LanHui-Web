## 1. 产品定位

蓝辉公开站不是单纯的品牌画册，也不是强推销落地页。它需要完成一个克制、可信的车主决策闭环：

```text
理解蓝辉做什么
  → 找到适合车型或场景的产品
  → 理解产品、施工与服务优势
  → 找到真实营业门店
  → 通过电话或微信咨询
```

页面应让用户自然得出“蓝辉专注新能源轻改、方案更贴合车型、产品与施工更完整”的判断，而不是由网站反复宣称“我们最好”。

## 2. 核心用户

### 2.1 新能源车主

- 不熟悉轻改产品，需要先按车型或需求理解方案。
- 关心是否适配、是否影响日常使用、施工是否可靠。
- 最终希望找到真实门店进行咨询或施工。

### 2.2 轻改爱好者

- 已知道产品名称，希望快速查看款式、参数与适配车型。
- 需要在产品线、车型专题和门店之间快速跳转。

### 2.3 搜索与内容用户

- 从搜索引擎或文章进入。
- 先解决一个具体问题，再进入产品或门店路径。

加盟商不是公开站后台用户；加盟业务、合同、权益、收益和进货继续留在小程序或独立系统。

## 4. 全局导航

### 4.1 桌面端

主导航保持六个明确入口：

1. 首页。
2. 产品中心。
3. 车型方案。
4. 门店网络。
5. 品牌资讯。
6. 关于蓝辉。

右侧保留一个主 CTA“查找门店”，一个次 CTA“微信咨询”。不在 Header 同时堆放三个以上转化按钮。

### 4.2 移动端

- 使用抽屉式导航。
- “查找门店”固定为菜单内第一转化入口。
- 当前路由有明确选中态。
- 打开菜单后锁定背景滚动，支持 ESC、遮罩和关闭按钮退出。

### 4.3 全局页脚

- 完整规格见 [FOOTER_SYSTEM_PRD_2026-06-22.md](./FOOTER_SYSTEM_PRD_2026-06-22.md)。
- 品牌定位和真实地址。
- 产品、门店、资讯、品牌快捷入口。
- 真实联系方式；未确认内容不得展示假号码。
- 备案信息按实际状态展示。
- 不显示虚构“全国覆盖”“服务用户数量”等数字。
- 增加“关注蓝辉”：小红书、抖音、视频号；只有真实有效账号才显示。
- 小红书/抖音优先使用官方主页外链，视频号无稳定网页 URL 时使用独立二维码弹层。

## 5. 统一页面表达

### 5.1 视觉语气

- 深色基底保留品牌的新能源和汽车质感。
- 橙色只承担主操作、等级或重点信息，不大面积铺满。
- 内容区需要保留足够明暗层级，避免所有页面都是相同黑色卡片网格。
- 图片优先展示真实产品、材质、细节、施工和门店，不使用无关豪车图库制造档次。
- 动效只用于状态变化、筛选反馈和轻微进入，不采用持续闪烁、数字滚动或强迫注意。

### 5.2 内容语气

- 先说用户场景，再说产品能力。
- 参数后必须说明它对使用体验的意义。
- 不贬低原车，不制造车主焦虑。
- 不使用无法证明的“第一、顶级、全网最低、100%”。
- CTA 使用“查看方案、了解产品、查找门店、咨询门店”。

### 5.3 页面共同骨架

除首页外，一级页面统一使用：

```text
Header
→ 面包屑或栏目标签
→ 页面标题与一句价值说明
→ 页面主要任务区
→ 证据或补充内容
→ 下一步 CTA
→ Footer
```

## 6. 首页 `/`

### 6.1 页面任务

首页要在一次自然滚动中回答：

1. 蓝辉是什么。
2. 蓝辉主要服务哪些新能源车和哪些轻改需求。
3. 为什么值得继续了解。
4. 到哪里看产品。
5. 到哪里找门店。

### 6.2 模块顺序

#### A. Hero

- 主标题：新能源车轻改与车身膜一站式服务。
- 辅助说明：聚焦车型适配、产品选择、专业施工与售后支持。
- 主 CTA：查看车型方案。
- 次 CTA：查找门店。
- 首屏图片使用真实代表车型或施工成品，不堆叠多个产品小图。

#### B. 按车型找方案

- 展示当前已有的小米、问界、极氪专题。
- 每张入口卡说明车型范围和可查看的方案类型。
- 专题不足时不虚构更多品牌入口。

#### C. 按需求找产品

按用户语言分组，而不是只按内部产品名称罗列：

- 上下车与便利：电动踏板。
- 外观与姿态：轮毂、底盘、改色膜。
- 隔热与隐私：汽车窗膜。
- 漆面防护：隐形车衣。
- 内饰与空间：木地板。

#### D. 蓝辉优势

六项优势采用“主张 + 证据入口”表达：

| 优势 | 页面证据 |
|---|---|
| 聚焦新能源轻改 | 车型专题与适配说明 |
| 车型专项方案 | 按车型组织的产品清单 |
| 源头供应链与品质 | 产品材质、规格和质量说明 |
| 一站式服务 | 多产品线和统一服务流程 |
| 标准门店与专业施工 | 真实门店资料与施工说明 |
| 技术与售后支持 | 服务流程、交付和售后说明 |

#### E. 服务流程

`了解需求 → 确认车型与方案 → 到店施工 → 交付与售后`

流程用于降低用户的不确定感，不承诺尚未建立的预约系统。

#### F. 真实门店入口

- 当前只有一家真实门店时，直接展示“顺德大良店”，不包装成全国网络。
- 同时提供“查看门店网络”，进入可扩展的省市查询结构。
- 门店数量来自真实 active 数据。

#### G. 最新内容

- 展示 3 条最新已发布文章。
- 内容主题优先覆盖产品选择、施工知识、门店动态。
- 无文章时隐藏区块，不使用占位假文章。

#### H. 结尾 CTA

提供“查看产品方案”和“查找门店”两条路径，不使用折扣倒计时。

## 7. 产品中心 `/product`

### 7.1 页面任务

同时满足“按车型找”和“按产品找”，避免用户进入后面对十多个同权重入口。

### 7.2 页面结构

1. Hero：说明可按车型或需求选择。
2. 车型专题：小米、问界、极氪。
3. 产品分类：便利、外观、隔热隐私、防护、内饰。
4. 选择帮助：不知道如何选择时，查看文章或咨询门店。
5. 门店 CTA。

### 7.3 产品详情与车型专题

- 产品详情回答“是什么、适合谁、主要差异、施工流程、注意事项”。
- 车型专题回答“这款车有哪些可选升级”，并链接回对应产品知识。
- 产品卡不得只显示产品名称和图标，至少包含场景说明。
- 图片状态为缺失或待审核时，必须使用明确占位，不显示破图。

## 8. 门店网络 `/agent`

门店网络是本次优先级最高的公开页面系统，完整规则见 [STORE_NETWORK_PRD_2026-06-21.md](./STORE_NETWORK_PRD_2026-06-21.md)。

页面核心不是展示“我们有很多店”，而是让用户快速、准确地完成：

```text
选择省份 → 选择城市 → 按等级筛选 → 比较门店 → 查看详情 → 联系门店
```

全国、省份、城市、详情四层页面都必须适配“当前只有一家门店”和“未来多省多等级门店”两种规模。

## 9. 品牌页面 `/brand`

### 9.1 页面任务

解释蓝辉为什么聚焦新能源轻改，以及产品、供应链、门店和服务如何组成完整能力。

### 9.2 页面结构

1. 品牌定位。
2. 服务对象与业务边界。
3. 六项核心能力。
4. 产品与门店关系。
5. 已验证的资质和真实历程。
6. 产品与门店 CTA。

资质页只展示可核实证书；品牌历程不编造年份、规模和里程碑。

## 10. 资讯 `/news`

### 10.1 内容分类

- 产品知识。
- 车型方案。
- 施工与保养。
- 品牌动态。
- 门店动态。

### 10.2 页面结构

- 列表页支持分类筛选和分页。
- 卡片包含封面、分类、标题、摘要和日期。
- 详情页包含正文、相关产品、相关门店或咨询入口。
- 只有 `published` 文章可见。
- 已撤回文章返回 404，不继续出现在相关推荐中。

## 11. 联系页面 `/contact`

- 当前以总部微信和真实门店电话为主。
- 不单独建设留资表单、预约系统或在线支付。
- 用户需要找具体门店时，引导至门店网络。
- 联系方式缺失时显示可理解的替代路径，不展示假号码。

## 12. 全局状态

| 状态 | 处理原则 |
|---|---|
| 页面加载 | 首屏结构稳定，不整页闪烁 |
| 图片缺失 | 使用品牌统一占位，保留比例 |
| 数据失败 | 说明暂时无法加载并提供重试或其他入口 |
| 内容为空 | 隐藏非必要模块或展示真实空状态 |
| 筛选无结果 | 保留筛选条件并提供一键清除 |
| 内容已下线 | 返回 404，不泄露后台资料 |
| 联系方式缺失 | 提供总部咨询或返回门店列表 |

## 13. 门店与文章后台契约

### 13.1 门店

- 仅 `active` 门店公开。
- Province、City、Store Level 由后台统一维护。
- 前台不得根据自由文本推断省份、城市或等级。
- `pending`、`suspended`、`terminated` 不进入列表、数量、SEO 或详情页。
- 官网数量、等级和排序必须来自同一份 active 数据。

### 13.2 文章

- 仅 `published` 文章公开。
- 首页和列表使用同一发布时间排序。
- 撤回后同步移除列表、详情、相关推荐和 sitemap。

## 14. 当前实现差距

以下内容是实施前必须处理的已知差距：

1. `src/lib/store.ts` 仍包含 7 家演示门店，与“当前只有顺德大良真实门店”冲突。
2. Prisma `Store` 当前只有 `isActive`，尚未实现 `pending/active/suspended/terminated` 四状态。
3. Prisma `Store` 尚无 `level` 和官网 `displayOrder`。
4. 当前门店电话包含固定电话格式，与后台已确认的 11 位数字规则冲突。
5. 当前全国页宣称覆盖多个省份，来源是演示数据，不能继续作为正式内容。
6. 当前门店卡片在多个路由内重复实现，后续容易出现字段和视觉漂移。

以上差距未解决前，不得以“前台门店网络已完成”作为验收结论。

## 15. 埋点

使用稳定语义事件，不为每个视觉区块创建独立事件名：

| 事件 | 触发 |
|---|---|
| `page_view` | 页面访问 |
| `product_view` | 产品或车型方案详情 |
| `store_view` | 门店详情 |
| `contact_click` | 电话或微信 |
| `article_view` | 文章详情 |

公共字段：`pathname`、`sourceArea`、实体 ID/slug、渠道。不得采集手机号、车牌、姓名等 PII。

## 16. 响应式与可访问性

- 验收视口：1440、768、390。
- 移动端优先保证查找、筛选、拨号和咨询。
- 所有触控目标不小于 44×44px。
- 筛选器、抽屉、弹窗和菜单支持键盘操作。
- 等级不能只用颜色表达。
- 图片有具体 alt；装饰图使用空 alt。
- 页面只有一个 H1，标题层级连续。

## 17. SEO 与性能

- 首页：Organization + WebSite。
- 产品聚合：ItemList。
- 产品详情：Product 或 Service，字段必须真实。
- 门店全国页：仅在有真实门店时输出 ItemList。
- 门店详情：LocalBusiness + BreadcrumbList。
- 资讯详情：Article + BreadcrumbList。
- 非公开实体返回 404 并从 sitemap 移除。
- 无 Postgres 环境下 `npm run build` 必须继续成功，但静态兜底不得包含虚假业务数据。
- 图片提供固定宽高，避免 CLS。

## 18. 验收标准

### 18.1 产品表达

- [ ] 首页能自然说明新能源轻改定位和六项优势。
- [ ] 首页同时提供车型、产品和门店三条入口。
- [ ] 页面不包含虚假数量、覆盖范围、资质或紧迫营销。
- [ ] 产品中心能清楚区分“按车型找”和“按需求找”。

### 18.2 门店闭环

- [ ] 用户可按省份、城市和等级找到 active 门店。
- [ ] 全国、省份、城市、详情四层页面信息一致。
- [ ] 当前只有一家门店时页面仍真实、完整且不显空洞。
- [ ] 暂停或终止门店不公开且 URL 返回 404。
- [ ] 电话与微信 CTA 使用真实联系方式。

### 18.3 工程验收

- [ ] 1440、768、390 三视口通过。
- [ ] 键盘路径和焦点状态通过。
- [ ] SEO metadata 与 JSON-LD 使用真实数据。
- [ ] 埋点事件符合统一字典。
- [ ] 无数据库构建成功。
- [ ] 页面不再读取演示门店作为正式兜底。

## 19. 非目标

- 加盟商登录或加盟管理。
- 在线预约、订单、支付。
- 地图定位、距离排序和经纬度。
- 用户评价、评分和评论。
- 产品后台 CMS。
- 复杂个性化推荐。
