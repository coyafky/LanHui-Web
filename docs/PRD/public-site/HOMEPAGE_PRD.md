## 1. 系统目标

首页用 5 个区块在首屏内完成"品牌曝光 → 信任建立 → 服务矩阵 → 产品入口 → 转化路径"的完整漏斗。让第一次进入网站的新能源车主在一次自然滚动中完成判断：

1. 蓝辉主要做什么。
2. 是否有适合自己车型或需求的产品。
3. 蓝辉的方案、供应链、施工和售后是否可信。
4. 可以从哪里继续了解。
5. 可以联系哪一家真实门店。

### 核心路径

```text
品牌定位 → 按车型或需求进入 → 用产品与服务证据建立信任 → 查看真实门店 → 咨询
```

### 1.1 范围

- ✅ 包含：Header（共享）+ Hero + WhyChooseUs + CoreServices + ProductsQuickEntry + Footer（共享）
- ✅ 包含：品牌总机电话咨询转化闭环
- ✅ 包含：车型专题入口
- ✅ 包含：真实门店、最新文章和服务流程的信任闭环
- ❌ 不包含：产品详情子页（归各产品 PRD）
- ❌ 不包含：微信小程序 / 商城 / 支付 / 客户案例库

## 4. 页面结构

### 4.1 Hero：一句话说清蓝辉

- 主标题：聚焦新能源轻改与车身膜
- 副标题：说明车型适配、产品供应、门店施工与售后支持的一体化服务
- 主 CTA：查看产品（跳转 `/product`）
- 次 CTA：查找门店（跳转 `/agent`）
- 背景图：高品质门店/施工场景图
- 不直接要求添加微信为首屏目标

### 4.2 品牌定位

- 聚焦新能源轻改与车身膜
- 服务新能源车主的便利、外观、防护、隔热和座舱需求
- 从顺德大良真实门店开始建立服务能力

### 4.3 我们解决什么问题

- 通用配件信息多，车型适配难判断
- 参数很多但车主难理解使用价值
- 产品选择、施工和售后常常彼此割裂

蓝辉对应提供车型确认、产品说明、门店交付和售后支持。

### 4.4 核心能力

1. 新能源车型聚焦
2. 车型专项方案
3. 产品和供应链
4. 多产品一站式组合
5. 标准门店与专业施工
6. 技术和售后支持

每项能力至少链接一个可验证页面。

### 4.5 产品与车型入口

- 6 大产品线入口卡片
- 车型专题入口（问界/小米/极氪）
- 不在首页复制完整产品表

### 4.6 信任闭环

- 真实门店展示和说明
- 最新品牌/门店动态
- 服务流程：从需求确认到售后支持
- CTA：查看产品、查找门店

## 5. UI / 交互规范

### 5.1 视觉规范

- **背景**：`bg-zinc-950` / `bg-black` / `bg-zinc-900`（卡片）
- **文字**：`text-white` / `text-zinc-300` / `text-zinc-400`
- **品牌色**：`orange-400/500` 强调 + `blue-400/500` 科技感
- **字体**：Geist Sans + 系统中文
- **圆角**：卡片 `rounded-2xl`

### 5.2 组件清单

## 7. 现有页面问题与改进方向（来自 06-22 规划）

- Hero 使用"800 亩制造厂区"等表达，但证据来源未在官网数据层固化
- 首屏主 CTA 直接要求添加企业微信，用户尚未理解产品价值
- "为什么选择我们""核心服务""产品快速入口"连续使用相似卡片网格，视觉节奏单一
- 首页没有车型专题入口，无法体现新能源车型专项方案优势
- 产品入口只按内部产品线组织，普通车主不一定知道自己需要什么产品
- 没有真实门店、最新文章和服务流程组成的完整信任闭环
