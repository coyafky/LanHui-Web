# 蓝辉 Footer 官方社媒入口 PRD

> **范围**：全站共享 Footer  
> **平台**：小红书、抖音、微信视频号  
> **上位文档**：[FOOTER_SYSTEM_PRD_2026-06-22.md](./FOOTER_SYSTEM_PRD_2026-06-22.md)  
> **版本**：v0.1  
> **状态**：规划中，未授权编码

## 1. 目标

在全站 Footer 中提供蓝辉官方社媒入口，让官网用户可以继续查看短视频、图文内容和日常更新，同时让官网与官方社媒账号形成稳定、可信的品牌关联。

这里是官网指向社媒账号的外链。若希望获得严格意义上的“反向链接”，还需要在小红书、抖音、视频号账号资料或内容中填写官网地址；该动作不属于网站代码范围。

本文只定义 Footer 中的社媒子模块；Footer 的品牌、导航、联系方式、备案、响应式和整体验收规则以上位 Footer 系统 PRD 为准。

## 2. Footer 位置

在现有 Footer 中新增“关注蓝辉”区域，推荐位置：

```text
品牌简介
快捷导航
产品/服务
关注蓝辉
联系方式
```

桌面端作为独立列或位于联系方式上方；移动端放在联系方式之前。

不在首页正文重复建设大型社媒内容墙。

## 3. 平台处理

### 3.1 小红书

- 有经过确认的官方主页 URL 时，点击后新窗口打开。
- 显示平台名称和官方账号名称。
- 可选展示“小红书号”，但不能显示未确认占位账号。

### 3.2 抖音

- 有稳定、可在网页访问的官方主页 URL 时直接外链。
- 如果只有抖音号或分享口令，点击后打开账号信息弹层，提供复制和二维码。
- 不把短期分享链接作为永久 Footer 链接。

### 3.3 微信视频号

- 视频号通常不具备稳定公开网页主页链接时，使用二维码弹层。
- 显示视频号名称和“微信扫码关注”。
- 二维码必须独立于企业微信咨询二维码，不能混用。

## 4. 显示规则

每个平台至少需要：

- `platform`。
- 显示名称。
- 官方账号名。
- 有效 URL 或二维码。
- `status=active`。

规则：

- 没有真实 URL 和二维码时不显示该平台。
- 不显示“待补充”“XXX账号”等占位文案。
- 平台图标必须有文字标签或可访问名称。
- 最多显示已确认的三项，不因版面需要补造平台。

## 5. 交互

### 外链

- 新窗口打开。
- `rel="me noopener noreferrer"`。
- 明确的可访问名称，例如“在小红书关注蓝辉轻改”。
- 外链图标提示用户将离开官网。

`rel="me"` 用于声明官网与官方账号的身份关联；是否被平台或搜索引擎采纳由外部平台决定。

### 二维码弹层

- 与企业微信咨询弹窗视觉一致，但属于“关注社媒”场景。
- 标题、账号名、二维码和操作提示清楚区分平台。
- 支持 ESC、遮罩、关闭按钮和焦点恢复。
- 二维码加载失败时提供账号名复制；若账号名也缺失则隐藏入口。

## 6. 数据方案

一期推荐使用集中静态配置，不立即增加数据库和后台：

```text
platform: xiaohongshu | douyin | wechat_channels
label
accountName
profileUrl?
qrImagePath?
status: active | disabled
order
```

原因：

- 官方社媒账号数量少且变化低频。
- 现阶段尚未提供正式账号 URL 和二维码。
- 为三个 Footer 链接建设完整 CMS 会增加不必要复杂度。

当总部需要频繁替换账号、管理多个门店账号或设置生效时间时，再复用 `ConsultationChannel` 的管理模式建立 `SocialProfile` 表。

社媒关注二维码不能存入 `ConsultationChannel`，因为“关注内容”和“客户咨询”是两种不同业务行为。

## 7. 埋点

建议增加语义事件 `social_click`：

| 字段 | 示例 |
|---|---|
| `platform` | `xiaohongshu/douyin/wechat_channels` |
| `sourceArea` | `footer` |
| `action` | `open_profile/open_qr/copy_account` |

社媒点击属于内容关注，不纳入 `contact_click` 咨询意向。

不采集用户在外部平台的后续行为。

## 8. SEO 与品牌一致性

- Organization JSON-LD 的 `sameAs` 只加入经过确认、可公开访问的主页 URL。
- 视频号只有二维码而没有公开 URL 时不进入 `sameAs`。
- 社媒账号名称、头像和品牌名称应与官网一致。
- 外部主页也应尽可能填写官网 URL，形成双向身份验证。

## 9. 当前实现差距

- Footer 当前只有导航、产品和联系方式，没有社媒入口。
- `brand.ts` 没有社媒账号字段。
- 仓库中没有小红书、抖音或视频号的正式 URL/二维码资产。
- 当前只有企业微信咨询二维码，不能作为视频号二维码使用。
- Analytics 尚无 `social_click`。

## 10. 验收

- [ ] Footer 显示所有已确认 active 官方社媒。
- [ ] 小红书、抖音外链安全打开。
- [ ] 视频号无稳定链接时通过独立二维码关注。
- [ ] 未配置平台不会显示占位入口。
- [ ] Organization `sameAs` 只包含真实公开 URL。
- [ ] 社媒点击与客户咨询事件分开统计。
- [ ] 390px 下社媒入口可识别、可点击且不挤压 Footer。
