# 蓝辉四类轻改项目页面 PRD

> **覆盖页面**：电动踏板、轮毂、底盘护板、内饰改装  
> **上位文档**：[PRODUCT_INFORMATION_ARCHITECTURE_PRD_2026-06-22.md](./PRODUCT_INFORMATION_ARCHITECTURE_PRD_2026-06-22.md)  
> **版本**：v0.1  
> **状态**：规划中，未授权编码

## 1. 项目页共同目标

每个轻改项目页都要完成：

```text
为什么会需要
→ 有哪些不同方向
→ 自己的车型是否适合
→ 选择时看什么
→ 安装和使用有什么边界
→ 哪些品牌车型已有方案
→ 到哪里确认
```

## 2. 电动踏板 `/product/electric-steps`

### 页面印象

`让高底盘新能源 SUV/MPV 上下车更方便，同时兼顾收起后的原车观感`

### 模块

1. Hero：展开/收起双状态视觉。
2. 典型场景：老人、儿童、裙装、频繁上下车。
3. 工作方式：开门展开、关门收起。
4. 选择维度：
   - 车型专用支架。
   - 踏板宽度与防滑。
   - 承重与结构。
   - 单/双电机。
   - 灯光与迎宾效果。
   - 与原车电气连接方式。
5. 安装边界：车型、年款、底盘结构、线束方案。
6. 已覆盖品牌：优先问界；理想、极氪、小米按真实数据显示。
7. 车型产品卡。
8. 使用与维护：清洁、涉水、异物和异常收起。
9. 施工流程。
10. 企业微信咨询与门店。

## 3. 轮毂 `/product/wheels`

### 页面印象

`在尺寸、数据和日常驾驶边界内，调整整车风格与姿态`

### 模块

1. Hero：同车型不同风格轮毂效果。
2. 风格方向：运动、豪华、简洁、越野。
3. 核心数据解释：
   - 直径和宽度。
   - PCD/孔距。
   - ET。
   - 中心孔。
   - 轮胎规格。
   - 刹车空间。
4. 参数对日常体验的影响。
5. 车型确认流程。
6. 四品牌轮毂入口；无真实数据的品牌显示“方案待补充”，不展示假款式。
7. 安装流程：检查、动平衡、扭矩、胎压和复检。
8. 合规与安全提示。
9. 相关文章和咨询。

页面不只写“款式多样、数据精准”四张价值卡。

## 4. 底盘护板 `/product/chassis-protection`

### 页面印象

`针对新能源车底部易受磕碰和泥水影响的区域，提供按车型确认的防护方案`

### 模块

1. Hero：车辆底部结构与护板位置示意。
2. 为什么需要：路面异物、托底、泥水、砂石。
3. 防护区域：
   - 前舱/电机区域。
   - 电池底部相关区域。
   - 中后段底盘。
   - 门槛和后护板。
4. 产品差异：
   - 材质。
   - 厚度。
   - 重量。
   - 散热和排水。
   - 固定点位。
   - 保养拆装便利。
5. 车型适配：现阶段重点问界，其他品牌按确认数据。
6. 安装检查：原车底部状态、孔位、异响、扭矩。
7. 使用边界：护板不等于绝对防撞，不进行夸大安全承诺。
8. 施工和复检流程。
9. 品牌车型入口与咨询。

旧 `/product/chassis` 的“底盘姿态/避震升级”与本页不同。建议后续重定向或另行保留“底盘操控”页面，不能混写。

## 5. 内饰改装 `/product/interior`

### 页面印象

`围绕座舱便利、空间整洁、触感保护和颜色协调进行车型化升级`

### 模块

1. Hero：完整座舱或后排空间效果。
2. 按使用场景：
   - 家庭出行。
   - 商务接待。
   - 收纳与整洁。
   - 高频接触保护。
   - 个性与颜色搭配。
3. 产品分类：
   - 地板总成与滑轨。
   - 尾箱地板和尾箱垫。
   - 坐垫、防踢垫和保护件。
   - 小桌板、支架、充电和收纳。
   - 方向盘、踏板、中控、出风口等细节。
4. 四品牌入口：
   - 极氪：座舱、尾箱和保护。
   - 小米：方向盘、中控、踏板和外观内饰细节。
   - 问界：地板、桌板、坐垫、收纳和后排便利。
   - 理想：地板、滑轨、脚踏、尾箱和家庭空间。
5. 颜色与内饰搭配。
6. 车型、年款和座椅布局适配。
7. 清洁维护与使用边界。
8. 地板专题等深度专题入口。
9. 咨询和门店。

## 6. 项目页数据要求

项目页不是一张静态说明页，需要聚合：

- 项目通用知识。
- 四品牌已确认产品。
- 相关车型项目。
- 相关文章。
- 公开咨询渠道。

项目页自身不建设产品 CMS，仍由静态产品数据和后续明确的数据源提供。
