## 1. 产品定位

“品牌资讯”升级为内容中心，但路由一期继续使用 `/news`，避免破坏现有链接和 SEO。

内容中心不是企业公告栏，而是通过持续内容帮助车主理解：

- 车型能做哪些升级。
- 产品应该怎样选择。
- 参数与实际体验有什么关系。
- 施工前后需要注意什么。
- 蓝辉有哪些真实品牌和门店动态。

## 2. 内容分类

一期采用五个固定分类：

| 分类 | 内容目的 |
|---|---|
| 车型方案 | 按车型梳理可选升级 |
| 产品知识 | 解释产品、参数和差异 |
| 施工与养护 | 施工流程、验收、使用与保养 |
| 门店动态 | 真实门店、服务和活动 |
| 品牌动态 | 企业、产品线和重要进展 |

不使用过细标签体系。标签继续作为可选元数据，不作为一期主要导航。

## 3. 内容列表 `/news`

### 3.1 页面结构

```text
Hero：内容中心定位
→ 重点文章
→ 分类筛选
→ 最新文章列表
→ 分页
→ 产品/门店 CTA
```

### 3.2 重点文章

- 最多一篇。
- 一期可沿用后台 `isSticky`。
- 显示封面、分类、标题、摘要、发布日期。
- 没有置顶文章时，直接从分类筛选开始。

### 3.3 分类筛选

- 默认“全部”。
- 筛选状态进入 URL：`/news?category=product-knowledge&page=2`。
- 切换分类时页码回到 1。
- 无结果时显示当前分类和“查看全部内容”。

### 3.4 文章卡片

- 封面图，可选。
- 分类。
- 标题。
- 摘要。
- 发布日期。
- 阅读入口。

不显示虚构阅读量。真实阅读量即使已采集，一期也不作为前台质量背书。

## 4. 文章详情 `/news/[slug]`

### 4.1 页面结构

```text
面包屑
→ 分类、日期、标题、摘要
→ 可选封面图
→ 正文
→ 相关产品/车型项目
→ 相关门店或统一咨询
→ 相关文章
```

### 4.2 正文阅读

- 正文宽度以阅读为主，不放入厚重卡片容器包住整篇文章。
- 支持标题、段落、列表、引用、图片和链接。
- 正文图片必须有 alt 和稳定比例。
- 长文可生成目录，但一期只有出现三个以上二级标题时显示。
- 移动端字号、行高和图片宽度优先保证阅读。

### 4.3 内容关联

一期不要求 CMS 建立复杂的多对多关系，可采用：

1. 编辑手工选择一个主要产品或车型项目；或
2. 暂时根据分类和正文内明确链接完成关联。

自动关键词推荐不进入一期。

### 4.4 状态

- `published`：公开。
- `draft`、`withdrawn`、`archived`：不公开。
- 已撤回文章返回 404，并从首页、列表、相关文章和 sitemap 移除。

## 5. 图片策略

- 封面图一期可选，避免增加发布负担。
- 有封面时建议 16:9，用于列表和社交分享。
- 无封面时使用纯排版卡片或统一分类视觉，不强制生成假图片。
- 正文配图来自 CMS 上传能力。
- 不从公众号外链直接热链图片。

## 6. SEO

- 列表页使用 CollectionPage。
- 详情页使用 Article + BreadcrumbList。
- title 使用文章标题。
- description 优先摘要，否则截取正文。
- `publishedAt` 和 `modifiedAt` 使用真实时间。
- OG 图优先文章封面，否则使用品牌默认图。

## 7. 埋点

- `page_view`：列表页。
- `article_view`：详情页。
- `product_view`：从文章进入产品或项目后记录。
- `contact_click`：文章结尾咨询。

不采集用户阅读正文内容、选中文本或个人信息。

## 8. 当前实现差距

- 当前列表只有文章纵向卡片，没有分类筛选、重点文章和封面。
- 当前详情正文被包在厚重卡片中，阅读感更像后台预览。
- “相关文章”实际只排除当前文章，没有按同分类筛选。
- 静态兜底文章只有摘要，没有完整正文，不能作为正式公开内容。
- 当前 `NewsItem` 与 Prisma Article 字段映射不完整，缺少封面、真实发布时间和稳定状态表达。

## 9. 验收

- [ ] 用户可按五个固定分类浏览文章。
- [ ] 列表支持置顶、分页、无封面和空状态。
- [ ] 详情页具有完整阅读体验。
- [ ] 文章可以连接到相关产品、车型项目或门店。
- [ ] 非 published 文章完全不公开。
- [ ] CMS 保持极简发布原则，封面图和复杂标签不强制。

## 10. 非目标

- 评论、点赞、收藏。
- 用户投稿。
- 全文搜索。
- 自动同步发布微信公众号。
- AI 自动生成文章。
