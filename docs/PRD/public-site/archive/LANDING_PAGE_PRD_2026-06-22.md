## 1. 页面使命

首页不是把所有业务缩成卡片摆在一起，而是让第一次进入网站的新能源车主在一次自然滚动中完成判断：

1. 蓝辉主要做什么。
2. 是否有适合自己车型或需求的产品。
3. 蓝辉的方案、供应链、施工和售后是否可信。
4. 可以从哪里继续了解。
5. 可以联系哪一家真实门店。

核心路径：

```text
品牌定位
→ 按车型或需求进入
→ 用产品与服务证据建立信任
→ 查看真实门店
→ 咨询
```

## 2. 现有页面问题

- Hero 使用“800 亩制造厂区”等表达，但证据来源和对外使用边界尚未在官网数据层固化。
- 首屏主 CTA 直接要求添加企业微信，用户尚未理解产品价值。
- “为什么选择我们”“核心服务”“产品快速入口”连续使用相似卡片网格，视觉节奏单一。
- 首页没有车型专题入口，无法体现新能源车型专项方案优势。
- 产品入口只按内部产品线组织，普通车主不一定知道自己需要什么产品。
- 没有真实门店、最新文章和服务流程组成的完整信任闭环。

## 3. 页面结构

### 3.1 Hero：一句话说清蓝辉

- Eyebrow：`新能源车轻改与车身膜`
- H1 建议：`让新能源车的升级，更适合车型，也更适合日常使用`
- 说明：蓝辉围绕车型适配、产品选择、专业施工与售后支持，提供轻改装备和车身膜服务。
- 主 CTA：`查看车型方案`
- 次 CTA：`按需求找产品`
- 三级文字入口：`查找门店`
- 视觉使用一张真实代表车型或施工成品图。

Hero 不显示未经验证的面积、销量、门店数或客户数。已确认的供应链能力放在后续证据区说明。

### 3.2 双入口：按车型 / 按需求

#### 按车型找方案

- 小米。
- 问界。
- 极氪。
- 后续仅在有真实内容时增加车型品牌。

每个入口显示车型范围、主要升级方向和代表图，不显示虚构“热门指数”。

#### 按需求找产品

| 用户需求 | 对应产品 |
|---|---|
| 上下车更方便 | 电动踏板 |
| 外观与姿态升级 | 轮毂、底盘、改色膜 |
| 隔热、隐私与防晒 | 汽车窗膜 |
| 漆面保护 | 隐形车衣 |
| 座舱与空间质感 | 地板总成 |
| 日常清洁与车辆养护 | 洗美项目 |

### 3.3 优势证据区

六项优势不采用六张同质化口号卡，而使用三个较完整的证据模块：

1. **懂车型**：车型专题、适配提示和年款确认。
2. **有完整产品与供应链**：产品线、材料、参数和品质信息。
3. **能完成交付**：真实门店、施工流程和售后说明。

每个模块必须链接到可以继续验证的产品、品牌、门店或文章页面。

### 3.4 代表产品与项目

- 展示 3—4 个代表内容，不平铺全部产品。
- 建议组合：一个便利类产品、一个膜类产品、一个车型项目、一个座舱项目。
- 卡片说明用户场景、车型范围和结果，不只显示产品名称。
- “查看全部产品”进入 `/product`。

### 3.5 服务流程

```text
了解需求
→ 确认车型、年款与原车状态
→ 选择产品与施工方案
→ 到店施工和交付
→ 使用与售后支持
```

不承诺尚未存在的在线预约、即时库存或固定交付时长。

### 3.6 真实门店

- 当前展示顺德大良店真实资料。
- 展示门店等级、地址摘要、营业时间和封面图。
- CTA：`查看门店详情`、`浏览门店网络`。
- 不使用演示门店形成虚假全国覆盖。

### 3.7 内容与博客

- 展示一篇重点文章和两篇最新文章。
- 优先内容：车型方案、产品选择、施工知识。
- 无封面文章使用统一内容占位，不显示破图。
- 无已发布文章时隐藏本区块。

### 3.8 结尾 CTA

- 主 CTA：`查找门店`
- 次 CTA：`微信咨询`
- 文案强调“先确认车型与需求”，不制造限时感。

## 4. 响应式

- 桌面端 Hero 可使用左右布局，但文字宽度不超过阅读舒适范围。
- 768px 下车型和需求入口变为两列。
- 390px 下所有入口单列，主 CTA 优先。
- 移动端不使用横向无限轮播承载主要内容。

## 5. 数据

- 品牌信息：`src/lib/brand.ts`。
- 产品线：`src/lib/products.ts`。
- 车型专题：各 `<topic>-products.ts`。
- 门店：公开 active Store 数据。
- 文章：最新 published Article。

首页不得复制一份独立的产品、门店或文章内容造成漂移。

## 6. 埋点

- `page_view`：首页访问。
- `product_view`：进入产品或车型项目后记录，不在首页曝光时记录。
- `contact_click`：微信咨询。
- `store_view`：进入门店详情。

通过 `sourceArea` 区分 `hero`、`vehicle_entry`、`need_entry`、`featured_content`、`store_section` 和 `footer_cta`。

## 7. 验收

- [ ] 首屏清楚表达新能源轻改定位。
- [ ] 用户可按车型和按需求两种方式继续。
- [ ] 六项优势由三个证据模块支撑，不是口号堆叠。
- [ ] 只展示真实门店和已发布文章。
- [ ] 首页不出现未经验证的经营数字。
- [ ] 1440、768、390 三视口阅读和 CTA 正常。
- [ ] 无数据库构建时使用真实静态兜底。

## 8. 非目标

- 自动车型识别。
- 在线报价、预约和支付。
- 个性化推荐算法。
- 用户评价和案例数量统计。
