## 1. 系统目标

蓝辉公开站需要帮助车主：

1. 理解蓝辉专注的新能源轻改服务。
2. 找到适合自己车型和使用场景的升级方案。
3. 了解蓝辉在产品、供应链、施工和售后方面的能力。
4. 查看真实营业门店并进行咨询。
5. 通过文章持续了解产品、工艺和案例。

## 2. 官网表达原则

### 2.1 克制营销

- 不直接照搬发布会、朋友圈或销售培训话术。
- 不使用恐吓、身份焦虑、冒犯和诱导失去理性的表达。
- 不制造虚假稀缺、倒计时或夸张折扣。
- 不使用无法证实的销量、客户数、施工数和资质数字。
- 不把“推销”作为页面主要感受。

### 2.2 价值表达

- 用真实场景解释产品价值。
- 把参数翻译为便利、舒适、安全、审美和个性化体验。
- 产品数据、资质、门店和案例必须可验证。
- CTA 使用“了解方案”“查看车型”“寻找门店”“咨询门店”等自然表达。

## 3. Landing Page 应表达的优势

1. 聚焦新能源轻改。
2. 新能源车型专项方案。
3. 源头供应链与品质保障。
4. 一站式轻改产品与服务。
5. 标准门店与专业施工。
6. 技术与售后支持。

优势不能只写口号，需要由产品、车型、工艺、门店和服务流程共同证明。

## 5. 核心用户路径

### 产品路径

```text
首页优势/车型入口 → 产品或专题 → 了解方案 → 查找门店/咨询
```

### 内容路径

```text
首页/资讯列表 → 文章详情 → 相关产品/门店 → 咨询
```

### 门店路径

```text
首页/导航 → 门店网络 → 省份 → 城市 → 门店详情 → 电话/微信
```

每个页面族需要独立 PRD 与 Design。

## 7. 后台数据契约

- 公开站仅展示 `active` 门店。
- 文章仅展示 `published`。
- 门店等级、地区和名称以后台数据为准。
- 暂停或终止门店不进入列表和公开详情。
- 埋点使用分析系统定义的稳定语义事件。

## 8. 全局状态

- 数据加载失败。
- 当前无真实门店。
- 省份尚未开通门店。
- 城市暂无营业门店。
- 图片缺失。
- 文章已撤回或不存在。
- 动态路由 404。
- 电话或微信信息尚未配置。

空状态必须诚实，不使用虚构内容填充。
