# 蓝辉公开门店网络 PRD

> **上位文档**：[PUBLIC_SITE_SYSTEM_PRD_2026-06-21.md](./PUBLIC_SITE_SYSTEM_PRD_2026-06-21.md)  
> **后台契约**：[STORE_MANAGEMENT_PRD_2026-06-21.md](../admin/archive/STORE_MANAGEMENT_PRD_2026-06-21.md)  
> **页面**：`/agent`、`/agent/[province]`、`/agent/[province]/[city]`、`/agent/store/[id]`  
> **版本**：v0.2（页面规格深化中）  
> **创建日期**：2026-06-21  
> **状态**：规划中，未授权编码

## 1. 系统目标

帮助车主按省份、城市和门店等级找到真实营业的蓝辉门店，并自然完成电话或微信咨询。

门店网络同时要支持：

- 当前只有少量门店时保持真实可信。
- 未来加盟门店增多后无需重做信息架构。
- 与总部后台门店状态、地区和等级完全一致。

## 2. 数据可见性

| 后台状态 | 公开列表 | 公开详情 |
|---|---:|---:|
| `pending` 待发布 | 否 | 否 |
| `active` 营业中 | 是 | 是 |
| `suspended` 暂停合作 | 否 | 否 |
| `terminated` 终止合作 | 否 | 否 |

公开站不能通过 URL 泄露非 active 门店资料。

## 3. 页面层级

```text
/agent
  └─ /agent/[province]
       └─ /agent/[province]/[city]
            └─ /agent/store/[id]
```

### 全国页 `/agent`

- Hero 使用“查找蓝辉门店”，不使用演示数据宣称全国覆盖。
- 显示真实营业门店数量和已开通省份数量。
- 筛选条包含省份、城市、门店等级。
- 城市选择依赖省份；未选择省份时不可选。
- 提供有 active 门店的省份快捷入口。
- 显示当前筛选条件和结果数量。
- 默认结果区域展示全部 active 门店。
- 当前只有一家门店时，将真实门店作为推荐结果，不渲染空洞的大型省份矩阵。

### 省份页

- 省份名称与营业门店数量。
- 有门店的城市入口。
- 省内门店列表。
- 门店等级筛选。
- 城市筛选和等级筛选同步到 URL。
- 省内无 active 门店时不显示虚假卡片，提供返回全国页和统一咨询。

### 城市页

- 城市名称与营业门店数量。
- 城市门店列表。
- 门店等级筛选。
- 无门店的真实空状态。
- 提供返回所属省份和查看省内其他城市的路径。

### 门店详情

- 门店名称与等级。
- 一张真实封面图或统一占位图。
- 完整地址。
- 11 位联系电话。
- 营业时间。
- 门店介绍。
- 电话和微信咨询。
- 所属省、市返回路径。
- 等级名称旁提供一句简短说明。
- 详情页不展示用户评分、销量和未经验证的施工数量。

## 3.1 全国页区块顺序

```text
Header
→ Hero：查找蓝辉门店 + 真实数量
→ 省/市/等级筛选条
→ 已开通省份快捷入口
→ 门店结果列表
→ 无结果或网络建设说明
→ 总部咨询
→ Footer
```

筛选是全国页的主要任务，必须出现在首个内容区，不能被品牌介绍或大面积宣传图压到首屏以下。

## 3.2 省份页区块顺序

```text
面包屑
→ 省份标题 + 真实门店/城市数量
→ 城市入口
→ 等级筛选
→ 省内门店结果
→ 其他省份入口
```

## 3.3 城市页区块顺序

```text
面包屑
→ 城市标题 + 真实门店数量
→ 等级筛选
→ 城市门店结果
→ 省内其他城市
```

## 3.4 门店详情页区块顺序

```text
面包屑
→ 门店名称 + 等级
→ 封面图与核心联系信息
→ 门店介绍
→ 服务与营业说明
→ 电话/微信 CTA
→ 同城或省内其他门店
```

## 4. 门店等级

等级由后台统一维护，前台只读取和展示。

推荐使用稳定编码与可调整的展示名称：

| 稳定编码 | 候选展示名称 | 排序 | 前台说明方向 |
|---|---|---:|---|
| `flagship` | 星辉旗舰店 | 1 | 完整形象、产品展示与施工服务 |
| `premium` | 星耀尊享店 | 2 | 较完整的产品与服务能力 |
| `specialty` | 星辰专营店 | 3 | 聚焦蓝辉指定产品与施工 |
| `member` | 星光会员店 | 4 | 蓝辉合作服务门店 |

正式名称仍需确认。前台必须提供等级说明，不能只显示陌生徽章。

等级不是用户评分，也不代表星级评价。页面不得用 1—5 颗星表现门店等级，以免与消费者评分混淆。

## 5. 查找与筛选

### 筛选维度

- 省份。
- 城市。
- 门店等级。

城市必须依赖省份；切换省份时清除原城市。

### URL 规则

全国页使用查询参数保存筛选：

```text
/agent?province=guangdong
/agent?province=guangdong&city=foshan
/agent?province=guangdong&city=foshan&level=flagship
```

省份和城市详情页使用路径表达地区，等级继续使用查询参数：

```text
/agent/guangdong?level=flagship
/agent/guangdong/foshan?level=flagship
```

无效的 level 参数忽略并回退到“全部等级”；城市不属于省份时返回 404。

### 结果规则

- 默认只显示 active 门店。
- 筛选条件同步到 URL。
- 筛选无结果时显示当前条件，并允许一键清除等级或城市。
- 不按距离排序，因为一期不采集经纬度。

### 排序

一期固定排序：

1. 后台官网 `displayOrder` 升序。
2. 门店等级排序。
3. 门店名称拼音或稳定 slug 升序。

不提供用户可见的“最近上线”排序，避免把合作时间误解为质量排序。官网排序规则与后台管理列表排序是两个概念。

## 5.1 筛选器交互

### 桌面端

- 使用一行组合筛选条。
- 省份、城市使用选择器。
- 等级使用下拉选择或紧凑单选组。
- 筛选后更新结果数量，不整页跳回顶部。
- 已选择条件显示为可移除标签。

### 移动端

- 顶部显示“筛选”按钮和当前结果数量。
- 点击后打开底部 Sheet。
- Sheet 内按省份、城市、等级排列。
- “查看 N 家门店”为主按钮，“重置”为次操作。
- 关闭后焦点返回筛选按钮。

## 5.2 省份入口策略

推荐采用两层表达：

1. 主页面只突出有 active 门店的省份，保证每个入口都能看到真实结果。
2. “选择省份”控件可以包含完整启用省份数据；无门店省份标记“暂未开通”，选择后进入真实空状态。

该策略仍需用户确认。

## 6. 当前无门店或少量门店

### 全国无门店

- 不显示虚假的“全国覆盖”。
- 展示品牌总部或“门店网络正在建设中”。
- 提供统一咨询入口。

### 某省无门店

- 明确“该省目前暂无营业门店”。
- 提供查看其他省份和品牌咨询。
- 不创建虚假门店卡片。

### 某城市无门店

- 返回省份页。
- 查看该省其他城市。
- 统一咨询。

### 当前只有一家门店

- 全国页可以突出“顺德大良店”作为当前服务门店。
- 门店数量使用真实数据，不刻意营造规模感。
- 保留省市筛选结构，为未来扩张准备。

## 7. 门店卡片

卡片显示：

- 封面图。
- 门店名称。
- 门店等级。
- 省市和完整地址摘要。
- 简短地址。
- 营业时间。
- “查看门店”。

列表卡不直接堆叠全部联系方式；详情页承担完整信息和咨询。

### 7.1 卡片信息优先级

```text
封面图
→ 等级徽标
→ 门店名称
→ 城市与地址
→ 营业时间
→ 查看详情
```

- 电话不作为整张卡片的主要文本，避免列表信息过密。
- 等级徽标必须带文字。
- 图片缺失时使用统一门店占位图，不使用无关汽车照片。
- 整卡可点击，内部不嵌套第二个相同目的链接。

### 7.2 少量门店布局

- 1 家门店：使用一张最大宽度受控的重点卡片，不强行拉伸成三列。
- 2 家门店：桌面两列。
- 3 家及以上：桌面三列。
- 移动端始终单列。

## 8. 详情页转化

自然 CTA：

- 电话咨询。
- 微信咨询。
- 查看该城市其他门店。
- 了解服务项目。

不使用紧迫、折扣或夸张营销文案。

### 8.1 联系方式

- 联系电话来自后台 11 位数字字段，展示层生成 `tel:`。
- 电话缺失时不显示拨号按钮。
- 微信采用总部统一微信还是门店独立微信仍待确认。
- CTA 点击后记录 `contact_click`，不得记录电话号码本身。

### 8.2 封面图

- 一期每店一张封面图。
- 推荐比例 4:3，列表和详情复用同一资产。
- 图片缺失不阻止 pending 保存；是否阻止 active 发布遵循后台 PRD。
- 替换图片后，前台应在缓存刷新后显示新图。

## 9. 埋点

使用分析系统语义事件：

- `store_view`：进入门店详情。
- `contact_click`：电话或微信。

字段包括：

- storeId。
- province。
- city。
- level。
- channel。
- sourceArea。

省份/城市列表曝光不直接算作门店兴趣。

## 10. SEO

- 全国页：门店网络 ItemList（有真实门店时）。
- 省、市页：BreadcrumbList。
- 详情页：LocalBusiness + BreadcrumbList。
- 非 active 门店返回 404，不建立索引。
- 标题和描述使用真实门店数量。

## 11. 响应式和可访问性

- 移动端筛选使用底部 Sheet。
- 卡片单列。
- 电话 CTA 触控目标 ≥44px。
- 等级有文字说明。
- 面包屑语义正确。
- 图片有门店名称 alt。

## 12. 与后台的统一契约

- Province/City 是独立主数据。
- Store 只关联有效省市。
- 门店等级为共享枚举。
- 公开站只读取 active。
- 状态改变后按约定刷新公开数据。
- 前台不自行修正后台错误数据。

### 12.1 公开 Store 最低契约

| 字段 | 用途 |
|---|---|
| `id`、`slug` | 路由与埋点 |
| `name` | 门店名称 |
| `status` | 只允许 `active` |
| `level` | 等级筛选与展示 |
| `provinceSlug/Label` | 省份路由与展示 |
| `citySlug/Label` | 城市路由与展示 |
| `address` | 详细地址 |
| `phone` | 11 位联系电话 |
| `businessHours` | 营业时间 |
| `description` | 门店介绍 |
| `imagePath` | 封面图 |
| `displayOrder` | 官网排序 |

### 12.2 当前实现差距

- 当前 Prisma `Store` 只有 `isActive`，没有四状态、等级和官网排序字段。
- 当前静态兜底包含多家演示门店，正式上线前必须移除或替换为真实数据。
- 当前静态门店电话包含固定电话，与已确认的 11 位号码规则不一致。
- 当前门店卡片在全国、省份和城市页面重复实现，需要统一组件和字段口径。

## 14. 开放问题

1. 四个门店等级名称是否正式采用发布会版本。
2. 全国页是否确认采用“突出已开通省份 + 完整省份选择器显示未开通状态”的双层方案。
3. 门店封面图是否作为 active 发布必填。
4. 微信咨询是总部统一微信还是门店独立微信。
