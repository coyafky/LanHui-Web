## 1. Footer 的产品使命

Footer 不是把 Header 导航再复制一遍，也不是放置所有产品链接的仓库。它需要在用户浏览完任意页面后完成四件事：

1. 再次确认蓝辉是什么品牌。
2. 提供下一步最常用的内容入口。
3. 提供真实有效的门店、咨询和社媒联系方式。
4. 展示必要的版权、备案和法律信息。

用户路径：

```text
页面内容结束
→ 继续查看车型/产品/文章
→ 查找门店或咨询
→ 关注官方社媒
```

## 2. 现有 Footer 问题

- “产品中心”直接遍历现有 `products[]`，产品体系扩展后会不断拉长。
- 仍使用旧产品结构，缺少四品牌车型、内饰改装和洗美项目。
- 地址、电话、营业时间和邮箱存在占位值。
- `lanhui@example.com` 可能被误认为真实品牌邮箱。
- ICP 和公安备案即使是“待备案”也会直接展示。
- 没有微信/企业微信咨询入口。
- 没有小红书、抖音和视频号入口。
- 没有明确的内容中心、车型方案和服务入口分组。
- 外链、二维码、埋点、错误状态和移动端折叠规则没有定义。

## 3. 信息架构

推荐采用“品牌区 + 四组链接 + 法务底栏”：

```text
Footer
├── 品牌与咨询
│   ├── Logo
│   ├── 一句话品牌定位
│   ├── 微信/企业微信咨询
│   └── 查找门店
├── 车型方案
│   ├── 极氪
│   ├── 小米
│   ├── 问界
│   └── 理想
├── 产品与服务
│   ├── 轻改项目
│   ├── 汽车膜系
│   ├── 洗美项目
│   └── 产品中心
├── 内容与品牌
│   ├── 内容中心
│   ├── 品牌介绍
│   ├── 品牌资质
│   └── 联系我们
├── 关注蓝辉
│   ├── 小红书
│   ├── 抖音
│   └── 视频号
└── 底栏
    ├── 版权
    ├── ICP 备案
    ├── 公安备案
    └── 必要法律链接
```

联系方式可放在品牌区下方或单独作为桌面端第五列，最终以布局密度决定。

## 4. 品牌与咨询区

### 4.1 Logo

- 使用全站统一 Logo。
- 点击返回首页。
- 有明确 `aria-label`。
- 不在 Footer 再增加第二套品牌图形。

### 4.2 品牌说明

建议控制在两句话以内：

> 蓝辉轻改聚焦新能源车型，为车主提供车型化轻改、汽车膜、内饰与洗美养护服务。

不得显示未经核实的企业规模、客户数量、全国覆盖或官方授权。

### 4.3 主行动

Footer 最多保留两个明显行动：

- `微信咨询`：打开统一咨询承接弹窗。
- `查找门店`：进入 `/agent`。

不在 Footer 同时增加电话、微信、预约、报价、导航等多个同等级按钮。

### 4.4 联系信息

仅显示已经确认的数据：

- 总部或当前服务门店名称。
- 真实地址。
- 真实联系电话。
- 营业时间。

规则：

- 电话必须可点击拨号。
- 地址链接至门店详情，不直接假装提供地图导航。
- 邮箱没有正式品牌邮箱时不显示。
- 营业时间未确认时不显示“待确认”，改为引导咨询或完全隐藏。
- 不显示假 400 电话。

## 5. 车型方案区

固定展示四个品牌入口：

| 文案 | 路由 |
|---|---|
| 极氪车型方案 | `/product/zeekr` |
| 小米车型方案 | `/product/xiaomi` |
| 问界车型方案 | `/product/wenjie` |
| 理想车型方案 | `/product/li-auto` |

如果某品牌页面尚未具备真实可公开内容：

- 不创建死链接。
- 可以暂时隐藏该入口。
- 不显示“即将上线”占位，除非已经确定上线计划。

Footer 不展开每个品牌下的全部车型。

## 6. 产品与服务区

Footer 只提供聚合入口，避免列出十几个页面：

| 文案 | 路由 | 说明 |
|---|---|---|
| 轻改项目 | `/product#light-mod` | 电动踏板、轮毂、底盘护板、内饰 |
| 汽车膜系 | `/product#film` | 窗膜、改色膜、隐形车衣 |
| 洗美养护 | `/product/detailing` | 洗车、美容、内饰与膜面养护 |
| 全部产品与方案 | `/product` | 聚合入口 |

若锚点没有稳定实现，则一期直接链接 `/product`，不可保留无效 hash。

## 7. 内容与品牌区

推荐入口：

- 内容中心 `/news`。
- 品牌介绍 `/brand`。
- 资质与能力 `/brand/certifications`。
- 门店网络 `/agent`。
- 联系我们 `/contact`。

品牌历程可放在品牌页面内部，不必为了链接数量单独出现在 Footer。

## 8. 官方社媒区

完整规则见 [FOOTER_SOCIAL_LINKS_PRD_2026-06-22.md](./FOOTER_SOCIAL_LINKS_PRD_2026-06-22.md)。

### 8.1 平台

- 小红书。
- 抖音。
- 微信视频号。

### 8.2 展示

- 使用图标 + 平台文字，不只显示难以识别的图标。
- 有稳定主页 URL 的平台使用安全外链。
- 视频号无稳定网页 URL 时使用独立关注二维码。
- 未配置或 disabled 的平台不渲染。
- 社媒二维码与企业微信咨询二维码完全分开。

## 9. 底栏与法务信息

### 9.1 版权

格式：

```text
© 2026 LANHUI 蓝辉轻改
```

年份可以按当前年份自动更新，也可以保留品牌起始年份至当前年份；不得把未经确认的成立年份作为事实扩散。

### 9.2 ICP 备案

- 只有获得真实备案号后显示。
- 有官方备案查询链接时可安全外链。
- 不显示“ICP备案号待备案”。

### 9.3 公安备案

- 只有取得真实公安备案号后显示。
- 显示符合规范的备案图标和链接。
- 未备案时完全隐藏。

### 9.4 法律与隐私

一期只有实际存在并完成内容审核的页面才显示：

- 隐私说明。
- 网站使用条款。

如果不存在用户表单、Cookie 个性化和营销追踪，仍需要根据实际行为说明分析数据和第三方链接，但不要创建空白法律页面。

## 10. 桌面布局

### 10.1 推荐布局

1440px：

```text
品牌与咨询（宽）
车型方案
产品与服务
内容与品牌
关注蓝辉/联系方式
```

- 使用 12 栅格。
- 品牌区占 3—4 列。
- 其他列根据内容占 2—3 列。
- 每列不超过 5 个主要链接。
- 社媒图标与联系方式可上下排列。

### 10.2 视觉层级

- Logo 和咨询 CTA 为第一层。
- 分组标题为第二层。
- 链接和联系信息为第三层。
- 版权备案为最低层。
- Footer 不再重复大型渐变、营销数字或产品图片。

## 11. 平板和移动端

### 11.1 768px

- 品牌区占满一行。
- 其余导航分两列。
- 咨询 CTA 保持容易触达。

### 11.2 390px

推荐顺序：

1. Logo 和品牌说明。
2. 微信咨询、查找门店。
3. 车型方案。
4. 产品与服务。
5. 内容与品牌。
6. 关注蓝辉。
7. 联系信息。
8. 版权和备案。

链接分组可以使用常开列表；不建议为了节省少量高度强制使用手风琴。若最终内容超过合理长度，再使用可访问的 Accordion。

社媒和 CTA 触控区域不小于 44×44px。

## 12. 数据契约

### 12.1 一期推荐

Footer 自身不建设数据库表，使用几个受控数据源：

| 数据 | 来源 |
|---|---|
| 品牌名称、说明 | `src/lib/brand.ts` 或后续品牌配置 |
| 产品与车型路由 | 明确的导航配置，不直接遍历全部产品 |
| 门店联系方式 | 公开 active Store 或总部真实配置 |
| 微信咨询 | ConsultationChannel 公开解析 |
| 社媒账号 | 集中 `socialProfiles` 静态配置 |
| 备案 | 真实部署/品牌配置 |

Footer 链接应由显式配置维护，不自动把数据库中的全部产品、文章或门店塞入 Footer。

### 12.2 推荐类型

```text
FooterLinkGroup
  title
  links[]

FooterLink
  label
  href
  external?
  status

SocialProfile
  platform
  accountName
  profileUrl?
  qrImagePath?
  status
  order
```

## 13. 页面状态与降级

| 场景 | Footer 行为 |
|---|---|
| 联系电话未配置 | 隐藏电话，不显示占位 |
| 地址未配置 | 隐藏地址，保留门店网络入口 |
| 企业微信无有效渠道 | 微信 CTA 降级到联系页 |
| 社媒未配置 | 不显示对应平台 |
| 社媒二维码加载失败 | 显示账号复制或隐藏入口 |
| 品牌子页面尚未上线 | 隐藏链接或指向稳定聚合页 |
| 备案未完成 | 隐藏备案字段 |
| 数据源失败 | 保留静态导航和品牌 Logo，不阻断页面 |

Footer 失败不能导致整页渲染失败。

## 14. 可访问性

- 使用语义化 `<footer>` 和分组标题。
- 导航区可使用 `<nav aria-label="页脚导航">`。
- 联系信息使用 `<address>`，但避免默认斜体。
- 外链有可访问名称和外链提示。
- 图标不能替代文字。
- 键盘焦点清楚。
- 二维码弹层遵循焦点管理规范。
- 文字对比度满足 WCAG AA。

## 15. SEO 与外链

- 内链使用稳定、可索引的真实路由。
- 不链接 draft、withdrawn、suspended 或不存在的实体。
- 官方社媒主页可使用 `rel="me noopener noreferrer"`。
- Organization JSON-LD `sameAs` 与 Footer 的有效公开社媒 URL 使用同一配置。
- 不在 Footer 堆叠 SEO 关键词或重复城市名。
- 链接文案描述目的，不使用大量“点击这里”。

## 16. 埋点

Footer 使用既有语义事件：

| 行为 | 事件 |
|---|---|
| 打开微信咨询 | `contact_click` |
| 点击电话 | `contact_click` |
| 点击门店 | 进入详情后记录 `store_view` |
| 点击小红书/抖音/视频号 | `social_click` |
| 普通内部导航 | 默认不作为经营核心事件 |

公共字段：

- `sourceArea=footer`。
- `channel` 或 `platform`。
- 可选目标 route/entity。

Footer 链接曝光不计为兴趣或咨询。

## 17. 当前实现迁移

### 保留

- 共享 `<Footer>`。
- Logo。
- 深色视觉基线。
- 内部链接使用 Next Link。
- 版权与备案底栏结构。

### 调整

- 移除直接 `products.map()` 生成全部产品链接。
- 新增显式车型、服务、内容和社媒分组。
- 电话和地址改用真实公开数据。
- 企业微信接入统一 ConsultationChannel。
- 社媒接入 `socialProfiles`。
- 未配置备案不渲染。

### 删除或禁止

- `lanhui@example.com` 等占位联系方式。
- `ICP备案号待备案`、`公安备案号待备案`。
- 无效产品路由。
- 假账号和未确认社媒链接。

## 18. 测试

### 功能

- 所有内部链接返回有效页面。
- 外链 URL 正确、安全打开。
- 企业微信咨询弹窗带 `sourceArea=footer`。
- 视频号二维码与咨询二维码不混用。
- 缺少电话、社媒、备案时正确隐藏。

### 响应式

- 1440：五区结构清楚，无过宽空白。
- 768：品牌全宽，其余两列。
- 390：顺序合理，无横向溢出。

### 可访问性

- 键盘遍历顺序合理。
- 外链和二维码按钮有清晰名称。
- 焦点可见。
- 图标隐藏后文字仍可理解。

### 回归

- 所有公开页面只渲染一个 Footer。
- Footer 不阻塞 SSG 和无数据库 build。
- 无社媒配置时不出现空白列。
- 长中文地址不会破坏布局。

## 19. 验收标准

- [ ] Footer 能清楚表达品牌定位和下一步行动。
- [ ] 车型、产品服务、内容品牌、社媒和联系方式分组明确。
- [ ] 不再自动平铺所有产品。
- [ ] 所有显示的电话、地址、邮箱、备案和社媒都是真实数据。
- [ ] 微信咨询、门店与社媒入口行为一致且可追踪。
- [ ] 未配置内容能够自然隐藏，不出现“待补充”。
- [ ] 1440、768、390 三视口通过。
- [ ] Footer 不复制 Landing Page 的大段营销内容。

## 20. 非目标

- Footer 内嵌社媒内容流。
- Footer 内展示文章列表或门店列表。
- Newsletter 邮件订阅。
- 多语言切换。
- 复杂 Footer CMS。
- 为占满列数虚构产品、联系方式或社媒账号。

## 21. 待确认

1. Footer 联系区最终展示总部信息，还是当前顺德大良店信息。
2. 是否已经具备可公开的隐私说明和网站条款页面。
