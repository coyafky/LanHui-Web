## 1. 页面定位

“洗美项目”是蓝辉产品中心中的独立服务项目，用于介绍车辆清洁、美容和日常养护能力。

它与轻改产品的区别是：

- 轻改解决车辆功能、外观和空间升级。
- 膜系解决隔热、改色和漆面保护。
- 洗美解决车辆日常清洁、漆面状态、内饰卫生和施工后养护。

页面不能只写“精洗、打蜡、镀晶”等服务名称，需要说明每种服务解决什么问题、适合什么车况、包含哪些步骤和不包含什么。

## 2. 页面主印象

`从日常清洁到漆面与内饰养护，让车辆保持干净、整洁和良好状态`

不使用“恢复新车”“百分百去除划痕”“永久保护”等无法保证的表达。

## 3. 页面模块

### 3.1 Hero

- H1：`汽车洗美与日常养护`
- 说明：围绕车身、漆面、玻璃、轮毂和内饰提供分级清洁与养护服务。
- 代表视觉：真实施工过程或完成后的车辆细节。
- CTA：`查看服务项目`、`咨询到店服务`。

### 3.2 按车辆状态选择

帮助用户先判断自己需要什么：

| 车辆状态 | 推荐方向 |
|---|---|
| 日常灰尘和轻度污渍 | 日常洗车 |
| 长期未深度清洁 | 全车精洗 |
| 内饰污渍、异味和缝隙积尘 | 内饰清洁 |
| 漆面失去光泽或有轻微太阳纹 | 漆面检查与美容 |
| 贴膜或车衣后的日常维护 | 膜面/车衣养护 |
| 轮毂、轮胎和玻璃污渍明显 | 局部专项清洁 |

### 3.3 服务分类

一期页面建议预留以下分类，实际公开项目必须以门店确认清单为准：

#### 日常洗车

- 车身预洗。
- 车身清洁。
- 轮毂和轮胎基础清洁。
- 擦干与基础检查。

#### 全车精洗

- 车身细节清洁。
- 门缝、标识、格栅等区域。
- 轮毂深度清洁。
- 内饰吸尘和表面清洁。

#### 内饰清洁

- 座椅、脚垫、地毯和缝隙。
- 中控、门板和高频接触区域。
- 材质适用性检查。
- 异味处理只说明实际方法，不承诺彻底消除。

#### 漆面美容

- 漆面状态检查。
- 去污、抛光或护理的适用边界。
- 轻微太阳纹与深划痕的区别。
- 打蜡、封体、镀晶等项目是否提供，待业务确认后公开。

#### 膜面与车衣养护

- 改色膜清洁。
- 隐形车衣膜面维护。
- 边角、污渍和表面状态检查。
- 不使用不适合膜面的清洁剂和施工方式。

#### 局部专项

- 玻璃。
- 轮毂。
- 轮胎。
- 虫胶、树胶和鸟粪等污渍。

### 3.4 服务详情卡

每个正式服务项目显示：

- 项目名称。
- 适合的车辆状态。
- 服务包含内容。
- 预计所需时间区间（有真实标准后展示）。
- 不包含事项。
- 施工前注意事项。
- 施工后维护建议。
- 可提供该服务的门店。

一期不显示未经确认的价格和促销。

### 3.5 洗美与其他产品的关系

- 贴窗膜、改色膜和车衣后：进入对应养护说明。
- 轻改施工前后：可提供车辆清洁和交付整理，但不得默认宣称免费。
- 内饰改装后：连接内饰清洁与维护。
- 品牌车型页：只作为通用养护入口，不为每个品牌复制一套洗美项目。

### 3.6 服务流程

```text
确认车辆状态
→ 说明服务内容与风险
→ 车辆检查和物品确认
→ 分区施工
→ 完成检查
→ 交付与养护提醒
```

### 3.7 真实效果与内容

- 有授权的施工前后图才展示对比。
- 图片需要尽量保持相同角度和光线，避免夸大。
- 没有真实案例时展示施工步骤和局部细节。
- 关联洗车、漆面、内饰和膜面养护文章。

### 3.8 门店与咨询

- 显示实际提供洗美服务的 active 门店。
- 企业微信咨询携带 `entityType=service`、`entityId=detailing`。
- 不允许用户误以为所有加盟门店都自动提供全部洗美服务。

## 4. 数据要求

一期产品线固定，可先由静态数据维护：

```text
serviceKey
name
category
summary
suitableFor[]
includes[]
excludes[]
durationText?
beforeCare[]
afterCare[]
image
availableStoreIds[]
status
order
```

`status` 至少区分 `draft/active`，只有 active 项目公开。

如果未来洗美服务、价格和门店供应频繁变化，再独立建设服务项目 CMS；本期不并入现有产品 CMS。

## 5. SEO 与埋点

- 页面类型：`service_detail`。
- Schema：Service + BreadcrumbList。
- 进入详情记录 `product_view`，metadata 使用 `contentType=service`、`serviceKey=detailing`。
- 咨询记录 `contact_click`。
- 进入门店详情记录 `store_view`。
