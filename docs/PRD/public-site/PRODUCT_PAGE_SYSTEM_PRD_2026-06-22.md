## 1. 产品页面使命

产品页面不是电子货架。它需要帮助车主从用车问题出发，理解产品价值、适配边界、选择差异和施工过程，再进入车型项目或门店咨询。

产品线一期仍固定，不建设产品 CMS。

## 2. 产品中心 `/product`

### 2.1 双入口

页面首个主要内容区提供：

- **按车型找方案**：进入车型项目页。
- **按需求找产品**：进入固定产品线。

车型专题和产品线不再作为十个同权重 Banner/卡片连续堆叠。

### 2.2 需求分类

| 需求组 | 产品 |
|---|---|
| 便利升级 | 电动踏板 |
| 外观与姿态 | 轮毂、底盘、改色膜 |
| 隔热与隐私 | 汽车窗膜 |
| 漆面防护 | 隐形车衣 |
| 座舱与空间 | 地板总成 |

### 2.3 产品卡片

- 使用真实产品或施工图；缺图时使用产品类别占位。
- 产品名称。
- 一句话使用场景。
- 适合的典型用户。
- “了解产品”。

不以图标和内部分类名作为卡片主要内容。

## 3. 产品详情页

六条固定产品线和地板产品采用统一内容逻辑，但允许膜类、轻改装备和地板保留必要差异。

### 3.1 页面回答顺序

1. 这是什么。
2. 它解决什么用车问题。
3. 哪些车主和车型适合。
4. 有哪些方案或系列差异。
5. 选择时要看哪些参数。
6. 怎样确认适配和施工。
7. 下一步去哪里。

### 3.2 标准页面结构

```text
面包屑
→ Hero：产品 + 场景价值 + 代表图
→ 适合谁 / 不一定适合谁
→ 核心价值
→ 产品系列、套餐或选择指南
→ 参数解释
→ 车型适配与风险提示
→ 施工与交付流程
→ 相关车型项目
→ 相关文章
→ 查找门店 / 咨询
```

### 3.3 产品差异

- 电动踏板：车型接口、展开收起、承重与安装方式。
- 轮毂：尺寸、孔距、ET、中心孔、轮胎与法规提示。
- 底盘：日常舒适、支撑、定位和非极端化边界。
- 汽车窗膜：可见光、紫外线、红外线、总太阳能阻隔和套餐差异。
- 改色膜：颜色、表面效果、施工和使用维护。
- 隐形车衣：材质、厚度、涂层、防护和质保。
- 地板总成：车型、座椅布局、颜色、组件和空间使用。

### 3.4 参数表达

- 原始参数保留。
- 每组参数后提供“这对用户意味着什么”。
- 未确认参数不展示。
- 质保、认证和性能数字必须有来源。

### 3.5 产品状态

产品线固定，但产品图片或款式可能存在：

- `matched`：图片与产品确认匹配。
- `pending-review`：待业务复核。
- `missing`：缺图。

前台只有 `matched` 可作为正式视觉；其余状态使用明确占位，不得假装产品实拍。

## 4. 产品与项目页关系

- 产品页按“产品能力”组织。
- 项目页按“车型完整方案”组织。
- 产品页链接适用车型项目。
- 项目页链接产品知识。
- 两者共享门店和咨询入口，不建立两套 CTA。

## 5. 当前实现差距

- 当前产品中心先连续展示四个专题 Banner，再展示两组产品，入口层级不够清楚。
- 多数产品卡片使用图标而不是真实产品视觉。
- 六条产品线共享 `ProductDetail`，但不同产品内部数据复杂度差异很大。
- 部分具体性能、质保和供应链文案需要核验来源。
- 产品详情与文章、车型项目、真实门店之间缺少稳定关联。

## 6. 验收

- [ ] 产品中心明确区分按车型和按需求。
- [ ] 每个产品详情完成问题—选择—适配—施工—咨询闭环。
- [ ] 参数配有用户意义解释。
- [ ] 未核实数字不上线。
- [ ] 产品与车型项目双向关联。
- [ ] 缺图不会造成破图或虚假实拍。

## 7. 非目标

- 产品后台 CMS。
- 在线库存、报价、下单和支付。
- 自动车型匹配。
- 车型法规合规承诺。
