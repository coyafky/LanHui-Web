## 2. 背景

蓝辉轻改当前产品中心主要展示 6 个通用产品方向：电动踏板、轮毂升级、底盘升级、窗膜、改色膜、隐形车衣。随着 MPV、新能源家庭车和商务接待车型的轻改需求增加，用户会围绕“地板总成 / 地板套件 / 尾箱地板 / 中门迎宾踏板 / 休息脚踏”等项目进行了解和对比。

本专题用于把地板相关产品独立成一个产品分类入口，让用户先理解地板改装能解决什么问题、有哪些结构组成、适合哪些车型场景。

这些素材来自“东魅商改画册”。页面级图片中可能出现供应商品牌、画册版式、价格和车型营销文案。

实现时必须遵守：

- 首版可使用页面级图片做专题视觉素材，但上线前必须由业务确认是否允许保留“东魅”品牌露出。
- 如果业务不允许供应商品牌露出，coder 需要从页面级图片中裁切产品区域，另存为 `public/images/products/flooring/crops/`，再用于正式页面。
- 价格信息默认不展示。若业务要求展示价格，必须在 PRD refine 后明确价格口径、有效期、门店差异和免责声明。

- 不得宣称“东魅官方合作”“厂家授权”“原厂件”“官方件”，除非业务提供书面授权或证明。

## 4. 用户与业务目标

### 4.1 用户角色

| 用户 | 需求 |
| --- | --- |
| MPV / 新能源家庭车车主 | 想看到地板改装后的座舱效果、清洁便利性、脚踏与尾箱联动效果。 |
| 商务接待车主 | 关注座舱质感、耐用性、迎宾踏板、滑轨和后排使用体验。 |
| 潜在改装用户 | 想先按自己的品牌/车型找到适配案例，再了解不同颜色和场景的展示效果。 |
| 门店销售/运营 | 需要统一页面解释地板总成的组成、卖点和施工流程。 |
| 品牌负责人 | 通过分类专题提升“蓝辉轻改在内饰地板升级方面专业”的感知。 |

### 4.2 业务结果

本轮要建立一个地板改装分类专题闭环：

1. 用户在 `/product` 看到“地板改装专题”入口。
2. 用户进入 `/product/flooring`。
3. 用户按适配品牌/车型找到相关地板套件案例。
4. 用户理解地板改装的典型组成、核心卖点、颜色效果和适配方向。

### 6.2 专题页结构

`/product/flooring` 页面建议结构：

1. Hero 区：专题名称、核心价值、代表视觉图。
2. 核心卖点区：座舱质感、回归空间、脚踏便利、滑轨整合、尾箱联动、颜色适配。
3. 结构组成区：地板主板、铝合金滑轨条、中门迎宾踏板、LED 休息脚踏、三排/尾箱地板。
4. 适配品牌/车型区：按品牌或车系分组展示不同车型的地板总成案例。
5. 颜色轮播区：同一车型下用轮播展示不同颜色或材质视觉，例如雪霜白、中性灰、岩石黑、木纹咖。
6. 车型/场景图库：按 MPV / 商务 / 家庭 / 新能源等场景补充展示图片。
7. 服务流程：车型确认、款式选择、安装评估、施工交付。

### 7.2 页面主卖点与辅助卖点

首屏只放 3 个主卖点，避免信息过载：

| 优先级 | 主卖点 | 首屏短文案 |
| ---: | --- | --- |
| 1 | 按车型适配 | `按品牌车型查看地板总成案例` |

| 3 | 套件化升级 | `地板、滑轨、脚踏、尾箱区域统一呈现` |

中段可展开 6 个辅助卖点：

- 座舱质感提升。
- 后排空间更整洁。
- 中门踏板与休息脚踏改善上下车和乘坐体验。
- 滑轨条与座椅轨道区域做视觉整合。
- 尾箱地板与后排地板形成统一效果。
- 不同颜色适配不同内饰风格。

本轮模板只保留以下热门品牌。其他品牌即使出现在 PDF 或素材池中，也不作为本轮页面编写模板。

| 品牌/车系 | 车型文案 | 推荐展示页 | 展示重点 |
| --- | --- | --- | --- |
| 理想 | 理想 L 系列 / 理想 MEGA 等业务确认车型 | 待业务补充或从素材池人工匹配 | 家庭出行、后排空间、尾箱联动、颜色适配。 |

素材使用约束：

- 如果某热门品牌没有对应实拍或可确认素材，前端不要借用其他品牌图片。
- 缺素材品牌可以先保留在数据模板中，但页面不展示误导性车型图。
- PDF 中其他品牌内容只作为卖点和结构参考，不进入本轮品牌模板。

### 7.5 颜色轮播规则

同一车型若 PDF 展示多种颜色，应使用轮播而不是重复创建多个车型卡片。

首版颜色命名：

| 颜色 ID | 展示名称 | 适用说明 |
| --- | --- | --- |
| `snow-white` | 雪霜白 | 浅色内饰、清爽明亮风格。 |
| `neutral-gray` | 中性灰 | 灰色/冷色内饰，偏克制商务。 |
| `rock-black` | 岩石黑 | 深色内饰，偏稳重耐脏视觉。 |
| `wood-brown` | 木纹咖 | 棕色、暖色或木纹风格内饰。 |

轮播要求：

- 每个车型模块最多首屏展示 1 张主图，颜色图进入轮播。
- 轮播卡片必须显示颜色名称和来源 PDF 页码。
- 价格信息不进入轮播文案。
- 如果某车型只有单色素材，仍保留车型模块，但不显示轮播控制。
- 颜色名来自画册可见内容或业务归纳，不能写成材质承诺。

### 7.6 价格口径

PDF 中可见部分画册价格，例如：

- 是否含税。
- 是否有车型、材质、颜色差异。
- 价格有效期。
- 是否允许展示供应商画册原价。

- 不在 JSX 中硬编码 55 个图片路径；应从静态数据数组或 manifest 派生。
- `assetPath` 使用 `/images/products/flooring/...`。
- 图片 alt 必须结合页面用途，例如：`地板改装套件页面素材 PDF 第 36 页`。
- `FlooringVehicleGroup` 是页面主体数据，不得把颜色变体建成独立车型或独立产品。
- `colorVariants` 只承载同一车型/套件的颜色视觉差异，不承载价格和材质承诺。
- `brand` 只能使用 `FlooringHotBrand` 中的五个值；不得把 PDF 中其他品牌加入本轮模板。

### 8.3 视觉与组件

- 使用 `next/image` 展示图片。
- 使用 shadcn/ui 的 `Button`、`Card`、`Badge`、`Table` 等组件时，只把它作为组件库。
- 颜色对比必须使用轮播或同等横向浏览组件，移动端必须可顺畅滑动。
- 品牌/车型分组建议使用锚点导航或分段筛选，不使用会隐藏大量内容的复杂客户端状态。
- 不要做营销落地页式大段空泛介绍，第一屏要直接显示“地板改装”产品视觉和分类价值。
- 不要把 55 张图一次性全尺寸首屏加载，应限制首屏图片数量，并对图库做 lazy loading。
- 不要使用纯图标替代真实产品图片。

### 8.4 页面动作与导航

专题页首版只承担分类展示与素材浏览，不新增固定动作区。

页面动作仅保留：

- 产品中心进入专题页的入口按钮。
- 专题页内部的分区浏览、图库查看更多或折叠展开。
- 页面导航使用站点已有导航组件，不在专题页新增独立返回按钮区。

### 8.5 SEO

`/product/flooring` 必须提供：

- `title`：`地板改装专题 | 蓝辉轻改 LANHUI`
- `description`：包含“地板总成、尾箱地板、迎宾踏板、MPV、新能源车型”等关键词。
- OpenGraph 图片：优先使用裁切后的专题封面；如果没有裁切封面，临时使用 `dongmei-flooring-page-36.jpg`，上线前需业务确认供应商品牌露出。
- JSON-LD：建议使用 `CollectionPage` 或 `ItemList`，不要写未经确认的价格。

## 11. 风险与待确认

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| PDF 没有可用文字层 | 无法自动抽取完整产品表 | 第一版使用页面级视觉素材，文案由 PRD 提供，后续人工补表。 |
| 页面级图片包含供应商品牌 | 可能影响蓝辉官网品牌一致性 | 上线前由业务确认；必要时裁切产品图。 |
| PDF 包含价格 | 价格可能过期或非蓝辉价格 | 首版不展示价格。 |
| 车型适配未形成结构化清单 | 用户无法按车型精确筛选 | 首版按场景展示，下一轮再补车型表。 |
| 材质描述未全部确认 | 可能形成材质承诺风险 | 只使用“地板套件”“铝合金滑轨条”等画册可见中性描述，避免夸大。 |
