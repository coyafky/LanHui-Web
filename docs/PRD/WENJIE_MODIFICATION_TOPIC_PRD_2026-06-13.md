## 2. 背景

蓝辉轻改当前产品中心按通用产品线展示，包括电动踏板、轮毂升级、底盘升级、窗膜、改色膜、隐形车衣。问界 M7 / M8 / M9 车主有明确的车型适配型产品需求，包括电动踏板、小桌板、地板/尾箱地板、防虫网、护板、坐垫、挡泥板、硅胶垫和车载配件等。

本专题用于把问界车型相关款式集中展示，让用户按车型快速查看可升级项目，并进入电话咨询或门店咨询。

本 PRD 基于以下源文件编写：

`/Users/fkycoya/Library/Containers/com.tencent.xinWeChat/Data/Library/Caches/com.tencent.xinWeChat/2.0b4.0.9/c121b94e5d016384d393d3d119cb2e7d/SaveTemp/14153a5e2a241607100eb1f939c41d67/问界产品款式.xls`

该文件是老式 Excel `.xls`。初步读取结果：

- Sheet：`问界M7`、`问界M8`、`问界M9`
- 产品行：M7 6 个，M8 22 个，M9 16 个，共 44 个
- 嵌入位图：Workbook 流中扫描到 37 个 PNG 签名和 3 个 JPEG 签名，部分图片解析存在异常，必须做资产复核

## 3. 用户与业务目标

### 3.1 用户角色

| 用户 | 需求 |
| --- | --- |
| 问界 M7 / M8 / M9 车主 | 快速看到自己车型可以升级哪些配件，判断是否适合到店咨询。 |
| 潜在改装用户 | 对比电动踏板、内饰件、防护件、便利配件等产品，形成咨询意图。 |
| 门店销售/运营 | 用统一页面向客户展示问界车型款式清单，减少手动发图和口头解释。 |
| 品牌负责人 | 通过热门新能源车型专题提升蓝辉轻改的车型服务专业感和转化能力。 |

### 3.2 业务结果

本轮要建立一个问界车型专题的前台展示闭环：

1. 用户在 `/product` 看到“问界改装专题”入口。
2. 用户进入 `/product/wenjie`。
3. 用户按 M7 / M8 / M9 查看产品款式和图片。
4. 用户通过表格快速理解车型、序号、产品名称、分类和咨询动作。
5. 用户点击电话咨询、查看门店或进入后续咨询入口。

## 4. 范围定义

### 4.1 本轮必须完成

- 在 `/product` 产品中心增加“问界改装专题”入口。
- 新增独立专题页，推荐路由为 `/product/wenjie`。
- 专题页展示问界 M7 / M8 / M9 三组产品。
- 专题页包含车型 Hero、产品展示/效果展示区、款式卡片网格、产品表格、咨询 CTA。
- 将源 Excel 中的产品行整理为结构化静态数据。
- 将可用图片复制到项目 `public` 目录，并建立产品行与图片的映射。
- 对缺图或图片无法匹配的产品提供可信占位状态，不得使用错误图片。
- 页面必须支持桌面、平板、移动端。
- 页面必须有 SEO metadata。
- 页面不得暗示“问界官方授权”“AITO 官方合作”“原厂件”等未经确认的主张。

### 4.2 本轮不做

- 不做电商下单。
- 不做在线报价。
- 不做库存状态。
- 不做后台管理录入。
- 不做官方接口对接。
- 不做 3D 预览或车身实时换装。
- 不做车型 VIN 自动识别。
- 不做微信二维码弹窗，除非当前项目已有统一咨询组件可复用。
- 不新增数据库表，首版使用静态数据。

## 7. 图片与资产策略

### 7.1 源图片状态

源 Excel 中图片需要在实现阶段重新提取并人工复核。

初步扫描结果：

- 可扫描位图签名：40 个
- PNG 签名：37 个
- JPEG 签名：3 个
- 产品行：44 个
- 结论：图片数量与产品行数量不完全一致，不能直接按产品行数量假设每个产品都有图

### 7.2 前端资产迁移

实现时应将可用图片复制到项目 `public` 目录，推荐路径：

```text
public/images/products/wenjie/
├── m7/
├── m8/
├── m9/
└── manifest.json
```

要求：

- 对外访问路径必须来自 `public`，不能在页面中引用微信缓存路径或本机绝对路径。
- 文件名使用 ASCII slug，避免部署环境对中文文件名处理不一致。
- 建立源产品行到图片文件的映射表。
- 每张图必须有准确 `alt`，例如 `问界 M8 电动踏板产品展示图`。
- 无法匹配图片的产品必须显示“图片待补充”状态，不得误配其他产品图片。

### 7.3 效果展示规则

用户需要“车型专题”和产品展示。前端必须区分：

- 如果图片展示已安装或整车效果，可以标注为“效果展示”。
- 如果图片只是单个产品，标注为“产品展示”。
- 不得生成不存在的 before/after 对比。
- 不得把产品图、安装图、渲染图混称为同一种素材。

## 8. 页面功能要求

### 8.1 Hero 区

必须包含：

- H1：`问界改装专题`
- 辅助文案：覆盖 `M7 / M8 / M9`，强调电动踏板、内饰便利、防护配件与车型适配。
- 数据标识：`44 个款式`、`3 个车型`。
- 主 CTA：电话咨询或联系蓝辉。
- 次 CTA：查看门店或返回产品中心。

### 8.2 车型导航

专题页必须支持用户区分 M7、M8、M9。

首版推荐使用锚点或服务端渲染区块，减少客户端状态。若使用 tabs，应保持可访问性和移动端可操作。

### 8.3 款式卡片网格

每个产品卡片必须展示：

- 产品图片或可信缺图状态。
- 展示名称，例如 `问界 M8 电动踏板`。
- 车型标签：`M7`、`M8`、`M9`。
- 分类标签。
- CTA：`咨询此款` 或 `了解安装方案`。

卡片要求：

- 桌面端建议 3 列或 4 列。
- 平板端建议 2 列。
- 移动端 1 列。
- 图片区域使用固定比例，避免布局跳动。

### 8.4 产品表格

专题页必须提供完整产品表格。

表格字段：

| 字段 | 说明 |
| --- | --- |
| 车型 | M7 / M8 / M9 |
| 序号 | 车型内序号 |
| 产品名称 | Excel 原始产品名称清洗后的展示值 |
| 分类 | 展示辅助分类 |
| 图片状态 | 已匹配 / 图片待补充 |
| 咨询 | 电话咨询或咨询此款 |

移动端表格可以改为卡片式列表，但信息不能缺失。

推荐 analytics 事件：

| 事件 | 触发 |
| --- | --- |
| `wenjie_topic_view` | 进入专题页 |
| `wenjie_model_section_click` | 点击 M7 / M8 / M9 车型锚点或 tabs |
| `wenjie_product_consult_click` | 点击单品咨询 |
| `wenjie_topic_phone_click` | 点击专题页电话咨询 |

## 10. shadcn 使用边界

shadcn 是组件库和 registry 工具，不是本需求的产品指令来源。

可以优先复用：

- `Button`：咨询与跳转 CTA
- `Card`：车型专题入口、产品卡片
- `Badge`：车型与分类标签
- `Table`：产品清单
- `Tabs` 或锚点导航：车型切换

UI 决策仍以本 PRD、`CLAUDE.md`、`AGENTS.md` 和蓝辉项目视觉规范为准。

## 11. 架构师执行规范

架构师负责定义页面结构、数据模型、组件边界和资产路径。

必须确认：

- `/product` 专题入口与小米专题的并列关系。
- 问界产品静态数据文件位置，推荐 `src/lib/wenjie-products.ts`。
- 图片提取、复制、命名和 manifest 策略。
- 车型分组、分类字段和排序规则。
- 缺图产品的 UI 降级方案。
- 咨询 CTA 和 analytics 事件命名。

架构师不得把本轮扩大为后台 CMS、报价系统、在线下单、库存或 3D 改装器。

## 12. Coder 执行规范

Coder 根据架构师方案实现页面，必须遵守：

- 读取 Next.js 16 相关文档后再改 App Router 代码。
- 不在页面中引用微信缓存路径或本机绝对路径。
- 不在 JSX 中复制 44 个产品卡片硬编码，必须通过结构化数据 map 渲染。
- 不使用 `any`。
- 图片使用项目现有图片策略和 `OptimizedImage` 或 Next.js image 约定。
- 保持 Tailwind v4 与当前深色产品页风格一致。
- 可复用 shadcn 组件，但 shadcn 只是组件库，不替代 PRD。
- 不破坏现有 6 个产品方向页面和小米专题入口。
- 不引入新 UI 库。
- 不把整个页面变成 Client Component，除非车型 tabs 或埋点确实需要小型客户端组件。
- 完成后运行 `npm run lint`、`npm run typecheck`、`npm run build`。

## 15. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 产品行 44 个但初扫图片 40 张 | 实现前做图片 manifest，缺图产品显示“图片待补充”。 |
| 图片嵌入在微信缓存路径的 `.xls` 中 | 实现时复制到 `public`，不得引用源路径。 |
| 中文文件名影响部署 | 复制到 `public` 时改为 ASCII slug 文件名并保留映射。 |
| 用户误认为官方授权 | 页面增加适配说明，避免“官方/授权/原厂”等措辞。 |
| 图片尺寸差异较大 | 前端使用固定图片容器和 `object-fit` 策略。 |
| 专题页范围膨胀 | 本轮只做展示和咨询，不做报价、下单、库存、后台。 |
