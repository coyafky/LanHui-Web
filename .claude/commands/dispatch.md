# /dispatch - LANHUI Agent Team Orchestrator

Use this command when a PRD or implementation plan is ready and the work should
be delegated across architect, coder, tester, and optional deployer roles.

## Role In The Workflow

`/dispatch` is the execution coordinator:

```text
approved PRD -> /plan -> /dispatch -> coder/tester/deployer reports -> /review -> /ship
```

If the input is still vague, run `/prompt-boost` first. If no PRD exists, run
`/spec` first.

## Required Inputs

One of:

- A PRD path under `docs/PRD/`.
- An implementation plan path under `docs/plans/`.
- A precise prompt generated by `/prompt-boost`.

## Agent Roles

- Architect: confirms file boundaries, data flow, tasks, risks, and acceptance criteria.
- Coder: implements approved tasks only.
- Tester: validates against PRD acceptance criteria and reports bugs.
- Deployer: runs build/deploy checks only when explicitly requested.
- Orchestrator: merges outputs, resolves conflicts, and makes the final decision.

## Worktree Rule

For multi-agent execution, each teammate must work in a separate worktree branch.
The orchestrator merges deliberately and reruns verification after merge.

## Process

1. Classify input:
   - vague request -> stop and run `/prompt-boost`
   - no PRD -> stop and run `/spec`
   - PRD but no implementation plan -> run `/plan`
   - approved plan -> dispatch agents
2. Give each agent only the context it needs:
   - PRD excerpt
   - plan task
   - relevant files
   - acceptance criteria
   - verification commands
3. Collect outputs.
4. Run quality gates:
   - `npm run lint`
   - `npm run typecheck`
   - `npm run build`
   - relevant tests and browser checks
5. Send unresolved issues back to the right role.
6. End with `/review`, then `/ship` when release-bound.

## Stop Conditions

Stop and ask the orchestrator when:

- Agents disagree on architecture.
- The PRD is ambiguous.
- Required assets or data are missing.
- A task introduces unverified official claims.
- Build, typecheck, or critical browser verification fails.

## Output

Return:

- Agent assignments.
- Worktree branches used.
- Files changed by each role.
- Verification results.
- Bugs found and resolution status.
- Recommended next command: `/review`, `/test`, or `/ship`.
